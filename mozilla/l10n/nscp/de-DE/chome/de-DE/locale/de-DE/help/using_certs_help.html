Beantragen eines eigenen Zertifikats

Wie ein Personalausweis oder eine Kreditkarte ist ein Zertifikat eine Form der Identifikation. Sie können sich damit über das Internet oder andere Netzwerke ausweisen. Zertifikate werden normalerweise von einer Organisation ausgestellt, die allgemein als Autorität für die Ausstellung solcher Ausweise anerkannt ist. Eine solche Organisation wird als Zertifikatsaussteller (ZA) bezeichnet.

Sie können Zertifikate zu Ihrer Identifikation bei öffentlichen ZAs, bei Systemadministratoren oder speziellen ZAs innerhalb Ihrer Organisation oder bei Websites beantragen, die spezielle Dienste anbieten, die eine zuverlässigere Form der Identifikation als über einen Benutzernamen und ein Passwort erfordern.

So wie die Anforderungen für das Ausstellen eines Führerscheins je nach Art des Fahrzeugs, das Sie fahren möchten, variieren, so hängen auch die Anforderungen für die Ausstellung eines Zertifikats davon ab, wofür Sie es verwenden möchten. In einigen Fällen brauchen Sie zum Beantragen eines Zertifikats lediglich einige persönliche Informationen auf einer Website anzugeben und können das Zertifikat automatisch in Ihren Browser herunterladen. In anderen Fällen müssen Sie kompliziertere Schritte ausführen.

Sie können noch heute ein Zertifikat beantragen, indem Sie die URL eines Zertifikatsausstellers aufsuchen und den Anleitungen auf dem Bildschirm folgen. Eine Liste von Zertifikatsausstellern finden Sie im Onlinedokument Client-Zertifikate.

Sobald Sie Ihr Zertifikat erhalten, wird es automatisch in einer Sicherheitseinrichtung gespeichert. Ihr Browser hat einen eigenen integrierten Software-Sicherheitsdienst. Eine Sicherheitseinrichtung kann auch ein Hardwaregerät, beispielsweise eine SmartCard, sein.

Wie ein Personalausweis oder eine Kreditkarte ist ein Zertifikat ein wichtiger Ausweis, der missbraucht werden kann, wenn er in die falschen Hände gerät. Sobald Sie ein Zertifikat erhalten haben, das Sie identifiziert, sollten Sie zwei Sicherheitsmaßnahmen ergreifen: eine Sicherheitskopie des Zertifikats anlegen und Ihr Master-Passwort einstellen.

Wenn Sie ein Zertifikat erhalten, werden Sie aufgefordert, es zu sichern. Wenn Sie bislang noch kein Master-Passwort eingegeben haben, werden Sie jetzt dazu aufgefordert.

Ausführliche Informationen zur Sicherung von Zertifikaten und zum Einstellen eines Master-Passworts finden Sie unter Ihre Zertifikate.

Überprüfen der Sicherheit für eine Website

Wenn Sie eine Website anzeigen, sehen Sie an dem Schlosssymbol in der unteren rechten Ecke des Fensters, ob der gesamte Inhalt der Website durch Verschlüsselung geschützt war, als Ihr Computer ihn empfing.

 Ein geschlossenes Schloss bedeutet, dass die Seite bei ihrem Empfang durch Verschlüsselung geschützt war.
   
 Ein offenes Schloss bedeutet, dass die Seite bei ihrem Empfang nicht durch Verschlüsselung geschützt war.
   
 Ein durchbrochenes Schloss bedeutet, dass einige oder alle Elemente der Seite beim Empfang nicht durch Verschlüsselung geschützt waren, obwohl die äußere HTML-Seite verschlüsselt war.

Weitere Informationen zum Verschlüsselungsstatus der Seite bei ihrem Empfang erhalten Sie, wenn Sie auf das Schlosssymbol klicken (oder im Menü Anzeigen auf Seiteninfo und dann auf die Registerkarte Sicherheit klicken).

Diese Registerkarte enthält zwei Arten von Informationen:

  • Der obere Teil des Fensters enthält Informationen darüber, ob die Website ausgewertet werden konnte. (Weitere Informationen zur Zertifikatsauswertung finden Sie unter Steuern der Auswertung.)
  • Der untere Teil beschreibt, ob der Inhalt der angezeigten Seite bei der Übermittlung im Netzwerk durch Verschlüsselung geschützt ist.

Wichtig:Das Schlosssymbol gibt nur den Verschlüsselungsstatus der Seite wieder, der gegeben war, als Ihr Computer die Seite empfing. Wenn Sie vor dem Senden oder Empfangen von unverschlüsselten Informationen benachrichtigt werden möchten, aktivieren Sie die entsprechenden SSL-Warnoptionen. Weitere Informationen finden Sie unter Einstellungen für Privatsphäre und Sicherheit – SSL.

Sicherheitseinrichtungen und -module

Der Geräte-Manager öffnet ein Fenster, in dem alle verfügbaren Sicherheitseinrichtungen aufgelistet sind. Mit dem Geräte-Manager können Sie alle Sicherheitseinrichtungen verwalten, einschließlich SmartCards, die Public Key Cryptography Standard (PKCS) #11 unterstützen.

Ein PKCS #11-Modul (auch als Sicherheitsmodul bezeichnet) steuert eine oder mehrere Sicherheitseinrichtungen, ähnlich, wie ein Softwaretreiber ein externes Gerät, beispielsweise einen Drucker oder ein Modem, steuert. Wenn Sie eine SmartCard installieren, müssen Sie das PKCS #11-Modul für die SmartCard auf Ihrem Computer installieren und den SmartCard-Leser anschließen.

Standardmäßig steuert der Geräte-Manager zwei interne PKCS #11-Module, die drei Sicherheitseinrichtungen verwalten.

  • Netscape Internal PKCS #11 Module: Das Modul steuert zwei Sicherheitseinrichtungen:
    • Allgemeiner Kryptografiedienst: Eine spezielle Sicherheitseinrichtung, die alle kryptografischen Operationen durchführt, die das Netscape Internal PKCS #11 Module erfordert.
    • Software-Sicherheitsdienst: Dieser Dienst speichert Ihre Zertifikate und Schlüssel, die nicht in externen Sicherheitseinrichtungen gespeichert sind, einschließlich aller ZA-Zertifikate, die Sie zusätzlich zu den im Lieferumfang des Browsers enthaltenen installiert haben.
  • Integriertes Roots-Modul: Dieses Modul steuert eine spezielle Sicherheitseinrichtung, die als integriertes Objekt-Token bezeichnet wird. Diese Sicherheitseinrichtung speichert die voreingestellten ZA-Zertifikate, die mit dem Browser geliefert werden.

Verwalten von SSL-Warnungen und -Einstellungen

Das Protokoll Secure Sockets Layer (SSL) ermöglicht Ihrem Computer, mit anderen Computern Informationen in verschlüsselter Form über das Internet auszutauschen, d. h. die Daten werden während der Übertragung so verändert, dass sie nicht von anderen gelesen werden können. SSL wird auch verwendet, um Computer im Internet mithilfe von Zertifikaten zu identifizieren.

Das Protokoll Transport Layer Security (TLS) ist ein neuer Standard, der auf SSL aufbaut. Standardmäßig unterstützt der Browser sowohl SSL als auch TLS. In den meisten Fällen funktioniert dies reibungslos, da gewährleistet ist, dass der Browser mit praktisch jeder anderen Software im Internet kompatibel ist, die eine beliebige Version von SSL oder TLS unterstützt.

Unter bestimmten Umständen kann es jedoch sein, dass Systemadministratoren oder andere Experten die SSL-Einstellungen an spezielle Sicherheitsanforderungen anpassen oder Fehler in einigen älteren Softwareprodukten kompensieren müssen.

Sie sollten die SSL-Einstellungen Ihres Browsers nur ändern, wenn Sie sich damit gut auskennen oder Unterstützung durch einen Fachmann haben. Sollten Sie die Einstellungen ändern müssen, gehen Sie folgendermaßen vor:

  1. Klicken Sie im Menü Bearbeiten auf Einstellungen.
  2. Klicken Sie in der Kategorie Privatsphäre und Sicherheit auf SSL. (Wenn in dieser Kategorie keine Einträge angezeigt werden, doppelklicken Sie auf Privatsphäre und Sicherheit, um die Liste zu erweitern.)

Steuern der Auswertung

Wie bereits unter Beantragen eines eigenen Zertifikats erläutert, ist ein Zertifikat eine Form der Identifikation, mit der Sie sich, ähnlich wie mit einem Personalausweis, über das Internet und andere Netzwerke ausweisen können. Ein Zertifikat kann aber ebenso wie ein Personalausweis ablaufen oder aus anderen Gründen ungültig werden. Ihr Browser muss daher die Gültigkeit jedes Zertifikats überprüfen, bevor er es als Identifikation akzeptiert.

Falls Ihnen die Verwendung von Zertifikaten nicht geläufig ist, sollten Sie Ihren Systemadministrator zurate ziehen, bevor Sie die Browsereinstellungen für die Zertifikatsauswertung ändern.

Funktionsweise der Auswertung

Jedes Mal, wenn Sie ein vom Zertifikats-Manager gespeichertes Zertifikat verwenden oder anzeigen, führt der Zertifikats-Manager verschiedene Schritte zur Überprüfung des Zertifikats durch. Zumindest überprüft er, ob die digitale Signatur des ZA auf dem Zertifikat von einem ZA stammt, dessen eigenes Zertifikat (1) in der Liste der verfügbaren ZA-Zertifikate enthalten ist und (2) als vertrauenswürdig für das Ausstellen der überprüften Zertifikatsart gekennzeichnet ist.

Wenn das ZA-Zertifikat nicht direkt vorhanden ist, muss in der Zertifikatskette für das ZA-Zertifikat ein höherstufiges ZA-Zertifikat vorhanden und als vertrauenswürdig gekennzeichnet sein. Der Zertifikats-Manager überprüft auch, ob das Zertifikat selbst als vertrauenswürdig gekennzeichnet ist. Wenn eine dieser Prüfungen fehlschlägt, markiert der Zertifikats-Manager dieses Zertifikat als unbestätigt und akzeptiert es nicht zur Identifikation.

Ein Zertifikat, das all diese Tests bestanden hat, kann dennoch in irgendeiner Weise fehlerhaft sein, beispielsweise kann es widerrufen worden sein, da eine unbefugte Person Zugang zu dem privaten Schlüssel des Zertifikats erlangt hat. Ein fehlerhaftes Zertifikat ermöglicht einer unbefugten Person (oder Website), sich als Zertifikatsinhaber auszugeben.

Diese Gefahr kann verhindert werden, indem das Zertifikat in einer Zertifikatsrücknahmeliste (Certificate Revocation List, CRL) als Bestandteil der Zertifikatsauswertung überprüft wird (siehe Verwalten von CRLs). Normalerweise laden Sie eine CRL in Ihren Browser, indem Sie auf eine Verknüpfung klicken. Wenn eine CRL vorhanden ist, überprüft der Zertifikats-Manager jedes von demselben ZA ausgestellte Zertifikat anhand der Liste.

Die Zuverlässigkeit der CLRs hängt von der Häufigkeit ab, mit der sie von einem Server aktualisiert und von einem Client überprüft werden. Sie können die Einstellungen für eine automatische CRL-Aktualisierung konfigurieren, sodass eine CRL in regelmäßigen Abständen automatisch mit der aktuell auf dem Server verfügbaren Version aktualisiert wird.

Eine andere Möglichkeit, der Gefahr fehlerhafter Zertifikate zu begegnen, ist die Verwendung eines speziellen Servers, der das Online Certificate Status Protocol (OCSP) unterstützt. Dieser Server kann dann Clientanfragen zu einzelnen Zertifikaten beantworten (siehe Konfigurieren von OCSP).

Der Server, ein so genannter OCSP-Responder, empfängt regelmäßig eine aktualisierte CRL von dem ZA, der die zu prüfenden Zertifikate ausgestellt hat. Sie können den Zertifikats-Manager so konfigurieren, dass er eine Statusanfrage zu einem Zertifikat an einen OCSP-Responder richtet, der dann überprüft, ob das Zertifikat gültig ist.

 

Verwalten von CRLs

Eine CRL (Certificate Revocation List, Zertifikatsrücknahmeliste) ist eine Liste widerrufener Zertifikate. Ein Zertifikatsaussteller (ZA) kann ein Zertifikat widerrufen, wenn es aus irgendeinem Grund fehlerhaft wurde – ähnlich wie ein Kreditkartenunternehmen Ihre Kreditkarte sperrt, wenn Sie diese als gestohlen melden.

Dieser Abschnitt beschreibt das Importieren und Verwalten von CRLs.

Weitere Informationen finden Sie unter Funktionsweise der Auswertung.

Eine genaue Beschreibung der CLR-Einstellungen, die Sie konfigurieren können, finden Sie unter Auswertungseinstellungen.

Datum der nächsten Aktualisierung

Der Browser verwendet die ihm zur Verfügung stehenden CRLs, um die Gültigkeit der von den entsprechenden ZAs ausgegebenen Zertifikate zu prüfen. Wenn ein Zertifikat als widerrufen in einer der Listen enthalten ist, akzeptiert der Browser es nicht zur Identifikation.

Ein ZA veröffentlicht normalerweise in regelmäßigen Abständen eine aktualisierte CRL. Jede CRL enthält ein Datum für die nächste Aktualisierung der CRL durch den ZA. Wenn dieses Datum älter als das aktuelle Tagesdatum ist, sollten Sie sich im Allgemeinen die neueste Version dieser CRL besorgen. Weitere Informationen zum Anzeigen der CRL-Informationen und Einrichten der automatischen CRL-Aktualisierung finden Sie unter Anzeigen und Verwalten von CRLs.

ZAs müssen vor dem Datum der nächsten Aktualisierung eine neue CRL vorlegen. Ein Zertifikat wird jedoch nicht allein deshalb ungültig, weil die aktuellste CLR fehlt. Deshalb kann ein Zertifikat als gültig ausgewertet werden, obwohl es im aktuellsten CRL als abgelaufen aufgeführt wird. Durch die automatische CRL-Aktualisierung kann dieser Fall verhindert werden.

 

Importieren von CRLs

Sie können die aktuellste CRL eines ZA in Ihren Browser importieren. So importieren Sie einen CRL:

  1. Rufen Sie die vom ZA oder Ihrem Systemadministrator angegebene URL auf und klicken Sie auf die Verknüpfung für die CRL, die Sie importieren möchten.

    Das Dialogfeld Status des CRL-Imports wird geöffnet.

  2. Bestätigen Sie, dass die CRL importiert wurde und es sich um die richtige handelt. In den meisten Fällen sollten Sie auf Ja klicken, wodurch die automatische Aktualisierung der gerade importierten CRL aktiviert wird.
  3. Der nächste Schritt hängt davon ab, ob Sie im Dialogfeld Status des CRL-Imports auf Ja oder Nein geklickt haben:

      Ja: Das Dialogfeld Einstellungen für automatische CRL-Aktualisierung wird angezeigt. Fahren Sie in diesem Fall mit Schritt 4 fort.

      Nein: Das Dialogfeld Status des CRL-Importswird geschlossen. Falls Sie Ihre Meinung ändern und doch noch die automatische Aktualisierung aktivieren möchten, finden Sie weitere Informationen unter Anzeigen und Verwalten von CRLs.

  4. Aktivieren Sie die Option Automatische Aktualisierung für diese CRL aktivieren.
  5. Legen Sie fest, in welchem Intervall die automatischen Aktualisierungen erfolgen sollen:
    • Aktualisieren __ Tag(e) bis zur nächsten Aktualisierung: Wählen Sie diese Option aus, wenn die Aktualisierung von der Häufigkeit abhängen soll, mit der die Herausgeber der CRL eine neue Version veröffentlichen.
    • Aktualisierung alle __ Tage: Wählen Sie diese Option aus, wenn die Aktualisierung unabhängig von der Aktualisierung der CRL erfolgen soll.
  6. Klicken Sie auf OK, um die ausgewählten Optionen zu bestätigen.

 

Anzeigen und Verwalten von CRLs

Sie können die CRLs, die dem Browser zur Verfügung stehen, im Fenster Auswertung anzeigen lassen und verwalten.

  1. Klicken Sie im Menü Bearbeiten auf Einstellungen.
  2. Klicken Sie in der Kategorie Privatsphäre und Sicherheit auf Auswertung. (Wenn in dieser Kategorie keine Einträge angezeigt werden, doppelklicken Sie auf Privatsphäre und Sicherheit, um die Liste zu erweitern.)
  3. Klicken Sie im Fenster Auswertung auf CRLs verwalten, um die CRLs anzuzeigen, die dem Zertifikats-Manager zur Verfügung stehen.

Um eine CRL zu löschen oder zu aktualisieren, wählen Sie sie aus und klicken Sie auf die entsprechende Schaltfläche.

Um die automatische Aktualisierung für eine CRL einzustellen, wählen Sie die CRL aus und klicken Sie auf Einstellungen. Das Dialogfeld Einstellungen für automatische CRL-Aktualisierung wird angezeigt.

  1. Aktivieren Sie die Option Automatische Aktualisierung für diese CRL aktivieren.
  2. Legen Sie fest, in welchem Intervall die automatischen Aktualisierungen erfolgen sollen:
    • Aktualisieren __ Tag(e) bis zur nächsten Aktualisierung. Wählen Sie diese Option aus, wenn die Aktualisierung von der Häufigkeit abhängen soll, mit der die Herausgeber der CLR eine neue Version veröffentlichen.
    • Aktualisierung alle __ Tage. Wählen Sie diese Option aus, wenn die Aktualisierung unabhängig von der Aktualisierung der CLR erfolgen soll.
  3. Klicken Sie auf OK, um die ausgewählten Optionen zu bestätigen.

 

Konfiguration von OCSP

Die Einstellungen für OCSP sind Teil der Auswertungseinstellungen. Um die Auswertungseinstellungen anzuzeigen, gehen Sie folgendermaßen vor:

  1. Klicken Sie im Menü Bearbeiten auf Einstellungen.
  2. Klicken Sie in der Kategorie Privatsphäre und Sicherheit auf Auswertung. (Wenn in dieser Kategorie keine Einträge angezeigt werden, doppelklicken Sie auf Privatsphäre und Sicherheit, um die Liste zu erweitern.)

Informationen über die verfügbaren OCSP-Optionen finden Sie unter OCSP.
