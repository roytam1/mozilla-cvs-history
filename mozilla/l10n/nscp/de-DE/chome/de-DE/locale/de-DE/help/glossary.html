Dieses Glossar wird nur zu Ihrer Information bereitgestellt und soll keine maßgebliche technische Beschreibung der erklärten Begriffe oder der komplexen Privatsphären- und Sicherheitsaspekte der erwähnten Technologien darstellen.

Glossar

Abhören. Unbefugtes Abfangen von Informationen, die über ein Netzwerk gesendet werden, durch eine Person, für die diese Informationen nicht bestimmt sind.

Adressleiste. Das Feld (und die zugehörigen Schaltflächen) oben in einem Navigator-Fenster, in das Sie eine URLoder Suchbegriff eingeben können. Weitere Informationen finden Sie unter Aufrufen einer anderen Seite.

Anfangsseite. Der Browser zeigt diese Seite nach jedem Starten oder Klicken auf die Schaltfläche Anfang an. Damit ist auch die Hauptseite einer Website gemeint, von der aus Sie die Site erkunden.

Ausdrückliche Zustimmung. Gelegentlich hört man den Begriff "Opt-out"-Zustimmung. Damit werden Privatsphäreneinstellungen bezeichnet, die den Websites das Sammeln von Informationen über Sie gestatten (z. B. mithilfe von Cookies und Onlineformularen), wenn Sie nicht ausdrücklich Ihre Zustimmung verweigern. Dazu wird auf der Seite eine Option bereitgestellt, die Sie aktivieren können. Ihre Zustimmung wird möglicherweise nicht dann angefordert, wenn die Informationen tatsächlich gesammelt werden.

Authentifizierung. Die Verwendung eines Passworts, Zertifikates, einer persönlichen Identifikationsnummer (PIN) oder anderer Informationen zur Bestätigung der Identität über ein Computernetzwerk. Siehe auch Authentifizierung durch Passwörter, Authentifizierung durch Zertifikate, Client-Authentifizierung, Server-Authentifizierung.

Authentifizierung durch Passwörter. Vertrauliche Identifikation mittels eines Namens und eines Passworts. Siehe auch Authentifizierung.

Authentifizierung durch Zertifikate. Überprüfung der Identität auf der Grundlage von Zertifikaten und Public-Key-Kryptografie. Siehe auch Authentifizierung durch Passwörter.

Cache. Eine Sammlung von Webseitenkopien, die auf der Festplatte oder im Arbeitsspeicher (RAM) des Computers gespeichert sind. Der Browser sammelt diese Kopien während Sie im Web surfen. Wenn Sie zum Aufrufen einer bestimmten Webseite, von der sich bereits eine Kopie im Cache befindet, auf eine Verknüpfung klicken oder eine URL eingeben, wird die Kopie mit dem Original verglichen. Hat sich nichts geändert, zeigt der Browser die gespeicherte Kopie an, statt die Originalseite aufzurufen, was die Zeit für die Verarbeitung und das Laden spart.

Client. Software (z. B. Browsersoftware), die Anfragen an einen Server sendet und von diesem Informationen empfängt. Einen Computer, auf dem Clientsoftware ausgeführt wird, bezeichnet man ebenfalls als Client.

Client-Authentifizierung. Identifikation eines Clientgegenüber einem Server, beispielsweise durch einen Namen und ein Passwort oder durch ein SSL-Client-Zertifikatund digital unterzeichnete Daten. Siehe auch Secure Sockets Layer (SSL), Server-Authentifizierung.

Cookie von Drittanbietern. Siehe Fremdes Cookie.

Cookie.Eine kleine Informationseinheit, die von einigen Websites auf Ihrem Computer gespeichert wird. Wenn Sie eine dieser Websites besuchen, veranlasst diese Ihren Browser, ein oder mehrere Cookies auf Ihrer Festplatte zu speichern. Beim nächsten Besuch der Website sendet Ihr Browser die entsprechenden Cookies zurück. Mithilfe von Cookies holen Websites bestimmte Informationen über Sie ein, wie beispielsweise den Inhalt Ihres Einkaufswagens. Sie können die Cookie-Einstellungen so festlegen, dass Sie deren Verwendung und die an Websites weitergegebene Datenmenge kontrollieren können. Siehe auch Fremder Cookie.

Cookie-Manager. Die Komponente des Browsers, mit deren Hilfe Sie die Verwendung von Cookies kontrollieren können. Weitere Informationen finden Sie unter Verwenden des Cookie-Managers.

Digitale ID. Siehe Zertifikat.

Digitale Signatur. Code, der aus den zu signierenden Daten und dem privaten Schlüssel des Unterzeichners erzeugt wird. Dieser Code ist für jeden neuen Datensatz einmalig. Selbst ein einziges Komma, das einer Nachricht hinzugefügt wird, ändert die digitale Signatur der Nachricht. Die erfolgreiche Bestätigung Ihrer digitalen Signatur durch eine entsprechende Software beweist nicht nur, dass Sie die Transaktion oder Nachricht genehmigt haben, sondern auch, dass die Daten nach dem Signieren nicht geändert wurden. Eine digitale Signatur hat nichts mit einer handschriftlichen Unterschrift zu tun, auch wenn Sie gelegentlich für dieselben rechtlichen Zwecke eingesetzt wird. Siehe auch Urheberschaftsbeleg, Manipulationserkennung.

Doppelte Schlüsselpaare. Zwei Paare aus je einem öffentlichen und privaten Schlüssel, insgesamt vier Schlüssel, die zwei separaten Zertifikaten entsprechen. Der private Schlüssel eines Paares wird für die Signaturoperationen verwendet und der öffentliche und private Schlüssel des anderen Paares werden für Ver- und Entschlüsselungsoperationen verwendet. Jedes Paar entspricht einem separaten Zertifikat. Siehe auch Public-Key-Kryptografie.

Entschlüsselung. Vorgang, bei dem verschlüsselte Daten wieder lesbar gemacht werden. Siehe auch Verschlüsselung.

Extensible Markup Language (XML). Ein offener Standard zur Beschreibung von Daten. Anders als bei HTML können Entwickler von Webseiten mithilfe von XML spezielle Tags definieren. Weitere Informationen finden Sie im W3C-Onlinedokument Extensible Markup Language (XML).

Extensible Stylesheet Language Transformation (XSLT). Eine Sprache, mit der XML-Dokumente in andere XML-Dokumente konvertiert werden. Informationen zur XSLT-Unterstützung von Netscape finden Sie unter Erweiterte Einstellungen – Erweitert.

Falschdarstellung. Selbstdarstellung einer Einrichtung als eine Person oder Organisation, die sie nicht ist. Eine Website kann beispielsweise vorgeben, ein Möbelhaus zu sein, während sie in Wirklichkeit nur Kreditkartenzahlungen einzieht, ohne Waren zu liefern. Siehe auch Spoofing.

File Transfer Protocol (FTP). Ein Standard, der Benutzern das Übertragen von Dateien zwischen Computern in einem Netzwerk ermöglicht. Sie können Ihren Browser verwenden, um Dateien mittels FTP abzurufen.

Fingerabdruck. Siehe Zertifikatsfingerabdruck.

FIPS PUBS 140-1. Federal Information Processing Standards Publications (FIPS PUBS) 140-1 ist ein Standard der US-Regierung für die Implementierung kryptografischer Module, d. h. von Hardware oder Software, die Daten ver- und entschlüsselt oder andere kryptografische Operationen (z. B. das Erstellen und Überprüfen digitaler Signaturen) durchführt. Viele Produkte, die an die US-Regierung verkauft werden, müssen einem oder mehreren FIPS-Standards entsprechen.

Formular-ManagerDie Komponente des Browsers, mit deren Hilfe Sie persönlichen Daten , die Sie in Onlineformulare eingeben, speichern, wie beispielsweise Namen, Adressen, Telefonnummern usw. Wenn eine Website ein Formular anzeigt, kann der Formular-Manager es automatisch ausfüllen. Weitere Informationen finden Sie unter Verwenden des Formular-Managers.

Fremdes Cookie. Ein Cookie von einer Website, das auf Ihrem Computer gespeichert wird, während Sie eine andere Website besuchen. Manchmal stellt eine Website Inhalte dar, die sich auf einer anderen Website befinden. Das kann alles Mögliche sein, von Grafiken bis hin zu Text oder Werbung. Die andere Website, die diese Elemente enthält, kann ebenfalls ein Cookie in Ihrem Browser hinterlassen, auch wenn Sie die Website nicht direkt besuchen. Diese Cookies werden auch "Cookies Dritter" genannt.

Hauptschlüssel. Symmetrischer Schlüssel, der vom Zertifikats-Manager zur Entschlüsselung von Informationen verwendet wird. Der Passwort-Manager beispielsweise verwendet den Zertifikats-Manager und Ihren Hauptschlüssel zur Verschlüsselung von eMail- und Website-Passwörtern und anderen gespeicherten, sicherheitsrelevanten Informationen. Siehe auch Symmetrische Verschlüsselung.

Hilfsprogramm. Jede Anwendung, die zum Öffnen oder Anzeigen einer vom Browser geladenen Datei verwendet wird. Ein Plugin ist eine spezielle Art von Hilfsprogramm, das sich selbst im Plugin-Verzeichnis des übergeordneten Installationsverzeichnisses des Browsers installiert, und in der Regel direkt im Browser geöffnet werden kann (intern). Microsoft Word, Adobe Photoshop und andere externe Anwendungen werden als Hilfsprogramme betrachtet, jedoch nicht als Plugins, da sie sich nicht im Browserverzeichnis installieren. Sie können jedoch im Download-Dialogfeld geöffnet werden.

Hypertext Markup Language (HTML). Das Dokumentformat von Webseiten. Der HTML-Standard definiert Tags bzw. Codes, mit denen das Textlayout, Schriften, Formate, Grafiken und andere Elemente definiert werden, aus denen die Webseite besteht.

Inhaber. Einrichtung (Person, Organisation oder Router), die von einem Zertifikat identifiziert wird. Das Inhaberfeld des Zertifikats enthält den Inhabernamen und andere Merkmale der Einrichtung.

Inhabername. Ein Unterscheidungsname, der den Inhabereines Zertifikats eindeutig beschreibt.

Internet Message Access Protocol (IMAP). Ein Standardprotokoll für Mail-Server, mit dem Sie alle Ihre Nachrichten und deren Änderungen auf dem Server speichern können, anstatt dies auf der Festplatte Ihres Computers zu tun. Durch die Verwendung von IMAP anstelle von POP sparen Sie Festplattenspeicher und Sie können von jedem Standort aus auf Ihre gesamte Mailbox zugreifen, einschließlich der gesendeten eMail, Entwürfe und der benutzerdefinierten Ordner. Einen IMAP-Server über ein Modem zu verwenden, ist in der Regel schneller als die Verwendung eines POP-Mail-Servers, da Sie zunächst nur die Nachrichtenkopfzeilen herunterladen. Nicht alle ISPs unterstützen IMAP.

Internet Protocol-Adresse (IP-Adresse). Die Adresse eines Computers in einem TCP/IP-Netzwerk. Jeder Computer im Internet besitzt eine IP-Adresse. Clients haben entweder eine permanente IP-Adresse oder eine Adresse, die ihnen bei jeder Verbindung mit dem Internet dynamisch zugewiesen wird. IP-Adressen werden in vier Zahlengruppen geschrieben: 204.171.64.2.

Internet. Ein weltweites Netzwerk aus Millionen von Computern, die über Standardprotokolle, wie dem TCP/IP miteinander kommunizieren. Es wurde 1969 ursprünglich für das US-Militär entwickelt, erstreckte sich dann auch auf Bildungs- und Forschungsinstitutionen, bis es in den späten 90er Jahren Millionen von Unternehmen, Organisationen und Einzelpersonen umfasste. Heute wird das Internet für das Versenden von eMail, das Surfen im World Wide Web, Instant Messaging, Diskussionsforen und viele andere Zwecke genutzt.

Java. Eine von Sun Microsystems entwickelte Programmiersprache. Ein einziges Java-Programm kann auf vielen verschiedenen Computern ausgeführt werden, weshalb Programmierer nicht mehr für jede Computerart ein eigenes Programm entwickeln müssen. Ihr Browser kann Java-Programme (auch Applets genannt) automatisch herunterladen und ausführen.

JavaScript. Eine Skriptsprache mit deren Hilfe Webseiten erstellt werden. Programmierer gestalten Webseiten mithilfe von JavaScript interaktiver, beispielsweise durch die Aufnahme von Formularen und Schaltflächen. JavaScript und Java können zusammen verwendet werden, sind jedoch technisch gesehen zwei unterschiedliche Sprachen. Für eine ordnungsgemäße Funktionsweise von JavaScript ist Java nicht erforderlich.

Kalender. Eine Kalenderanwendung zur Verwaltung Ihrer Termine und für ähnliche Aufgaben. Wenn Sie Netscape am Arbeitsplatz verwenden, können Sie vermutlich über das Menü Fenster darauf zugreifen. Falls Sie Netscape zu Hause verwenden oder an Ihrem Arbeitsplatz Netscape Calendar nicht installiert ist, erhalten Sie den webbasierten Kalender unter www.netscape.com.

Kodierschlüssel. Privater Schlüssel, der nur zur Verschlüsselung von Informationen verwendet wird. Ein Kodierschlüssel und der dazugehörige öffentliche Schlüssel sowie ein Signaturschlüssel und dessen öffentlicher Schlüssel ergeben zusammen ein doppeltes Schlüsselpaar.

Komponentenleiste. Die Symbolleiste links unten in einem Netscape-Fenster. Mithilfe der Komponentenleiste können Sie zwischen den Netscape-Komponenten wechseln, indem Sie jeweils auf die Symbole für Navigator, eMail und Diskussionsforen, Instant Messenger usw. klicken.

Kryptografie. Technik der Verschlüsselung und Entschlüsselung von Informationen. Kryptografische Verfahren werden beispielsweise eingesetzt, um Informationen, die zwischen kommerziellen Websites und Ihrem Browser ausgetauscht werden, zu ver- und wieder zu entschlüsseln. Siehe auch Public-Key-Kryptografie.

Kryptografischer Algorithmus. Regelwerk für die Durchführung kryptografischer Operationen wie Verschlüsselung und Entschlüsselung. Manchmal auch als Schlüssel bezeichnet.

Lesezeichen. Eine gespeicherte Webseitenadresse (URL), zu der Sie einfach wechseln können, indem Sie auf ein Lesezeichensymbol auf der Persönlichen Symbolleiste oder auf den Namen des Lesezeichens im Menü Lesezeichen klicken.

Lightweight Directory Access Protocol (LDAP). Ein Standardprotokoll für den plattformübergreifenden Zugriff auf Verzeichnisdienste, wie beispielsweise Unternehmensadressbücher. Sie können den Browser so einrichten, dass er über das Adressbuch auf LDAP-Verzeichnisse zugreift. Sie können auch eMail & Diskussionsforen für die Verwendung eines LDAP-Verzeichnisses zur automatischen Vervollständigung von eMail-Adressen einrichten.

Manipulationserkennung. Mechanismus, der erkennt, ob Daten in elektronischer Form während der Übertragung manipuliert wurden. Der Mechanismus überprüft, ob die empfangenen Daten vollständig mit den ursprünglichen Daten übereinstimmen.

Master-Passwort. Ein vom Zertifikats-Manager benutztes Passwort zum Schutz des Hauptschlüssels und/oder der privaten Schlüssel, die in einer Sicherheitseinrichtung gespeichert sind. Der Zertifikats-Manager muss auf Ihre privaten Schlüssel beispielsweise dann zugreifen, wenn Sie eMail-Nachrichten unterzeichnen oder eines Ihrer eigenen Zertifikate benutzen, um sich einer Website gegenüber zu identifizieren. Er muss auf Ihren Hauptschlüssel zugreifen, wenn der Passwort-Manager oder der Formular-Manager Ihre persönlichen Informationen lesen oder diese ergänzen. Sie können Ihr Master-Passwort im Einstellungsfenster Master-Passwörterfestlegen oder ändern. Jede Sicherheitseinrichtung erfordert ein eigenes Master-Passwort. Siehe auch Privater Schlüssel, Hauptschlüssel.

Navigationsleiste. Die Symbolleiste oben im Browserfenster, die die Vor- und Zurück-Schaltflächen enthält.

Netscape Quality Feedback Agent. Eine kleine Software-Einheit, die in Netscape 7.0 integriert ist und technische Informationen bei einem unerwarteten Programmabbruch sammelt. Nachdem Sie Ihre Erlaubnis gegeben haben, sendet der Agent diese Informationen an Netscape. Wenn Sie etwas dazu beitragen möchten, können Sie einen Kommentar mit einer Schilderung des Vorfalls hinzufügen. Netscape verwendet dieses Feedback zur Problembehebung und Verbesserung der Leistung. Die mit dem Agent verbundenen Dienste werden gelegentlich auch "Talkback" genannt. Weitere Informationen finden Sie im Onlinedokument Netscape Quality Feedback System for Netscape.

Objektsignatur. Technologie, die Software-Entwicklern das Signieren von Java-Code, JavaScripts oder anderen Dateien ermöglicht, sodass der Benutzer den Unterzeichner identifizieren und den Zugriff des signierten Codes auf lokale Systemressourcen kontrollieren kann.

Objektsignaturzertifikat. Zertifikat, dessen privater Schlüssel zum Signieren von Objekten wie Codedateien benutzt wird. Siehe auch Objektsignatur.

Öffentlicher Schlüssel. Ein Teil eines Schlüsselpaares, das in der Public-Key-Kryptografie verwendet wird. Der öffentliche Schlüssel ist frei zugänglich und wird als Teil eines Zertifikats ausgegeben. Er wird normalerweise für die Verschlüsselung von Daten benutzt, die an den Inhaber des öffentlichen Schlüssels gesendet werden, der die Daten mit dem entsprechenden privaten Schlüssel wieder entschlüsseln kann.

Online Certificate Status Protocol (OCSP). Regelwerk, das der Zertifikats-Manager befolgt, um jedes Mal, wenn ein Zertifikat verwendet wird, eine Onlineprüfung der Gültigkeit des Zertifikats durchzuführen. Bei diesem Vorgang wird das Zertifikat anhand einer Liste gültiger Zertifikate überprüft, die sich auf einer bestimmten Website befindet. Ihr Computer muss online sein, damit OCSP funktioniert.

Passwort-Manager.Die Browserkomponente, die Ihnen dabei hilft, Ihre Benutzernamen und Passwörter auf der Festplatte Ihres Computers zu speichern. Wenn Sie jetzt eine Website besuchen, für die diese Daten erforderlich sind, werden sie automatisch eingetragen. Weitere Informationen finden Sie unter Verwenden des Passwort-Managers.

Persönliche Symbolleiste. Diese Symbolleiste wird in Navigator standardmäßig unter der Adressleiste angezeigt und kann individuell angepasst werden. Sie enthält Standardschaltflächen wie Anfang, Suche, Lesezeichen usw., die Sie hinzufügen oder entfernen können. Sie können auch Schaltflächen für Ihre Lieblingslesezeichen oder Ordner mit Gruppen von Lesezeichen erstellen. Weitere Informationen finden Sie unter Persönliche Symbolleiste.

PKCS #11. Standard für Public-Key-Kryptografie, der Sicherheitseinrichtungen wie SmartCards regelt. Siehe auch Sicherheitseinrichtung, SmartCard.

PKCS #11-Modul. Programm auf Ihrem Computer, das kryptografische Dienste wie Ver- und Entschlüsselung unter Verwendung des Standards PKCS #11 verwaltet. Auch als kryptografische Module, kryptografische Dienste oder Sicherheitsmodulebezeichnet. PKCS #11-Module steuern Hardware- oder Softwaregeräte. Ein PKCS #11-Modul steuert einen oder mehrere Steckplätze, die als physische Lesegeräte (z. B. SmartCard-Leser) oder als Software implementiert sein können. Jeder Steckplatz eines PKCS #11-Moduls kann eine Sicherheitseinrichtung (auch Token genannt) enthalten. Dabei handelt es sich um ein Hardware- oder Softwaregerät, das kryptografische Dienste bereitstellt und Zertifikate und Schlüssel speichert. Der Zertifikats-Manager verfügt über zwei integrierte PKCS #11-Module. Sie können zusätzliche Module für die Steuerung von SmartCard-Lesern oder anderen Hardwaregeräten installieren.

Platform for Privacy Preferences (P3P). (Plattform für Privatsphäreneinstellungen) Ein Standard des World Wide Web Consortium (W3C), der dazu entwickelt wurde, dass Benutzer die Verwendung ihrer persönlichen Informationen von besuchten Websites besser kontrollieren können. Weitere Informationen zur Verwendung von Cookies in Websites, die diesen Standard unterstützen, finden Sie unter Festlegen der Privatsphärenebene. Allgemeine Informationen zu dem Standard finden Sie im Onlinedokument P3P Public Overview.

Plugin. Eine Art Hilfsprogramm, das dem Browser neue Funktionen hinzufügt, wie beispielsweise die Möglichkeit zum Abspielen von Audio- oder Videoclips. Im Unterschied zu anderen Hilfsprogrammen installiert sich ein Plugin selbst im Plugin-Verzeichnis des übergeordneten Installationsverzeichnisses des Browsers und kann in der Regel direkt im Browser geöffnet werden (intern). Ein Audio-Plugin beispielsweise ermöglicht es Ihnen, auf einer Webseite oder in einer eMail-Nachricht Audiodateien zu hören. Macromedia Flash Player und Java beispielsweise sind Plugin-Anwendungen.

Post Office Protocol (POP). Ein Standardprotokoll für Mail-Server, bei dem Sie neue Nachrichten auf der lokalen Festplatte speichern müssen. Allerdings haben Sie auch die Möglichkeit, Kopien auf dem Server zu belassen. Bei POP können Sie all Ihre Nachrichten, einschließlich gesendeter eMails, Entwürfe und benutzerdefinierter Ordner, lediglich auf einem Computer speichern. Im Gegensatz dazu ermöglicht Ihnen IMAP, alle Ihre Nachrichten und deren Änderungen auf dem Server zu speichern und darauf von jedem beliebigen Computer aus zuzugreifen. Die meisten ISPs unterstützen derzeit POP.

Privater Schlüssel. Ein Teil eines Schlüsselpaares, das in der Public-Key-Kryptografie verwendet wird. Der private Schlüssel ist geheim und wird zur Entschlüsselung von Daten benutzt, die mit dem entsprechenden öffentlichen Schlüssel verschlüsselt wurden.

Proxy. Eine Art Vermittlungsprogramm zwischen einem Serverund einem Client, das Anforderungen für andere Clients ausführt.

Public-Key-Infrastruktur (PKI). Die Standards und Dienste, die die Verwendung von Public-Key-Kryptografie und Zertifikaten in einer Netzwerkumgebung ermöglichen.

Public-Key-Kryptografie. Eine Sammlung etablierter Verfahren und Standards, die einer Einrichtung (einer Person, Organisation oder einem Hardwaregerät) die elektronische Bestätigung ihrer Identität oder die Signatur und Verschlüsselung elektronischer Daten ermöglicht. Dabei werden zwei Schlüssel benutzt: ein öffentlicher Schlüsselund ein privater Schlüssel. Der öffentliche Schlüssel wird als Teil eines Zertifikats ausgegeben, das diesen Schlüssel mit einer bestimmten Identität verknüpft. Der entsprechende private Schlüssel ist geheim. Daten, die mit dem öffentlichen Schlüssel verschlüsselt wurden, können nur mit dem privaten Schlüssel entschlüsselt werden.

Schlüssel. Siehe Kryptografischer Algorithmus.

Schlüssel. Eine lange Nummer, die von einem kryptografischen Algorithmuszur Ver- oder Entschlüsselung von Daten verwendet wird. Der öffentliche Schlüssel einer Person beispielsweise ermöglicht anderen, die Nachrichten dieser Person zu entschlüsseln. Die verschlüsselten Nachrichten müssen mit dem entsprechenden privaten Schlüssel entschlüsselt werden. Siehe auch Public-Key-Kryptografie.

Secure Sockets Layer (SSL). Protokoll, das die wechselseitige Authentifizierung zwischen einem Client und einem Server für den Aufbau einer verschlüsselten Verbindung ermöglicht. SSL läuft über TCP/IP und unter HTTP, LDAP, IMAP, NNTP und anderen Netzwerkprotokollen höherer Ebene. Transport Layer Security, der neue Standard der Internet Engineering Task Force (IETF), basiert auf SSL. Siehe auch Authentifizierung, Verschlüsselung.

Server. Software (z. B. für die Präsentation von Webseiten), die Anfragen von einem Client auf einem anderen Computer empfängt und diesem Informationen sendet. Ein Computer, auf dem Server-Software betrieben wird, bezeichnet man ebenfalls als Server.

Server-Authentifizierung. Identifikation eines Serversgegenüber einem Clientunter Verwendung eines SSL-Server-Zertifikats. Siehe auch Client-Authentifizierung, Secure Sockets Layer (SSL).

Sicherheitseinrichtung. Hard- oder Software, die kryptografische Dienste wie Ver- und Entschlüsselung bereitstellt und Zertifikate und Schlüssel speichert. Eine SmartCard ist ein Beispiel für eine Hardware-Sicherheitseinrichtung. Der Zertifikats-Manager enthält eine eigene Sicherheitseinrichtung, die als Software-Sicherheitsdienst bezeichnet wird und ständig verfügbar ist, solange der Browser aktiv ist. Jede Sicherheitseinrichtung ist durch ein eigenes Master-Passwort geschützt.

Sicherheitsmodul. Siehe PKCS #11-Modul.

Sicherheits-Token. Siehe Sicherheitseinrichtung.

Sicherheitszertifikat. Siehe Zertifikat.

Signaturschlüssel. Privater Schlüssel, der nur zum Signieren von Informationen verwendet wird. Ein Signaturschlüssel und der dazugehörige öffentliche Schlüssel sowie ein Kodierschlüsselund dessen öffentlicher Schlüssel ergeben zusammen ein doppeltes Schlüsselpaar.

Signaturzertifikat. Zertifikat, dessen privater Schlüssel zum Signieren übertragener Daten dient, damit der Empfänger die Identität des Absenders überprüfen kann. Zertifikatsaussteller (ZAs) geben ein Zertifikat zum Signieren von eMail-Nachrichten oft zusammen mit einem Verschlüsselungszertifikat aus, mit dem eMail-Nachrichten verschlüsselt werden können. Siehe auch Doppelte Schlüsselpaare, Digitale Signatur.

SmartCard. Gerät von der Größe einer Kreditkarte, das einen Mikroprozessor enthält und kryptografische Informationen (z. B. Schlüssel und Zertifikate) speichern und kryptografische Operationen durchführen kann. SmartCards verwenden den Standard PKCS #11. Eine SmartCard ist ein Beispiel für eine Sicherheitseinrichtung.

Software-Sicherheitsdienst. Standard-Sicherheitseinrichtung, die der Zertifikats-Manager zum Speichern der mit Ihren Zertifikaten verbundenen privaten Schlüssel verwendet. Neben den privaten Schlüsseln speichert der Software-Sicherheitsdienst den vom Passwort-Manager zur Verschlüsselung von eMail- und Website-Passwörtern und anderen sicherheitsrelevanten Informationen benutzten Hauptschlüssel. Siehe auch Privater Schlüssel und Hauptschlüssel.

Spoofing. Sich als eine andere Person ausgeben. Eine Person kann beispielsweise vorgeben, die eMail-Adresse jdoe@mozilla.com zu haben, oder ein Computer identifiziert sich selbst als Website namens www.mozilla.com, ohne es zu sein. Dies ist eine spezielle Form der Falschdarstellung.

SSL. Siehe Secure Sockets Layer (SSL).

SSL-Client-Zertifikat. Zertifikat, das ein Client(z. B. eine Browsersoftware) einem Server zur Bestätigung der Identität des Client (oder der Person, die den Client verwendet) unter Verwendung des Secure Sockets Layer (SSL)-Protokolls vorlegt. Siehe auch Client-Authentifizierung.

SSL-Server-Zertifikat. Zertifikat, das ein Servereinem Clientzur Bestätigung seiner Identität unter Verwendung des Secure Sockets Layer (SSL)-Protokolls vorlegt.

Stamm-ZA. Der Zertifikatsaussteller (ZA)mit einem selbst signierten Zertifikat, der an der Spitze einer Zertifikatskettesteht. Siehe auch Untergeordneter ZA.

Statusleiste. Die Symbolleiste unten in einem Netscape-Fenster. Auf der linken Seite enthält Sie die Komponentenleiste und auf der rechten Seite die Statussymbole. In Navigator enthält sie beispielsweise Symbole, wie das Schlosssymbol.

Steckplatz. Hardware oder eine entsprechende Software, die von einem PKCS #11-Modul gesteuert wird und zur Aufnahme einer Sicherheitseinrichtung dient.

Suchmaschine. Eine Webanwendung, die Benutzern das Suchen und Abrufen bestimmter Informationen im World Wide Web ermöglicht. Die Suchmaschine kann den gesamten Text von Webdokumenten oder Liste von Schlüsselwörtern durchsuchen. Es ist aber auch möglich, dasWebdokumente von Personen überprüft und manuell für den Abruf indiziert werden. In der Regel gibt der Benutzer ein Wort oder einen Satz in ein Suchfeld ein und die Suchmaschine zeigt dann Verknüpfungen zu den relevanten Webseiten an.

Symmetrische Verschlüsselung. Verschlüsselungsmethode, die eine Nachricht mit demselben Schlüssel ver- und entschlüsselt.

Talkback. Siehe Netscape Quality Feedback Agent.

TCP. Siehe Transmission Control Protocol/Internet Protocol (TCP/IP).

TLS. Siehe Secure Sockets Layer (SSL).

Token. Siehe Sicherheitseinrichtung.

Transmission Control Protocol/Internet Protocol (TCP/IP). Ein Protokoll von Unix, mit dem Computer, auf denen unterschiedliche Betriebssysteme ausgeführt werden, verbunden werden können. TCP/IP ist ein wichtiges Internetprotokoll und gehört mittlerweile zu den weltweiten Standards.

Uniform Resource Locator (URL). Die Standardadresse anhand der der Browser erkennt, wie er eine Datei oder andere Ressource im Web findet. Beispiel: http://www.netscape.de. Sie können die URL in die Adressleiste des Browsers eingeben, um auf Webseiten zuzugreifen. URLs werden auch in den Verknüpfungen auf Webseiten verwendet, auf die Sie klicken, um zu anderen Webseiten zu wechseln. Die URL wird auch Internetadresse oder Webadresse genannt.

Untergeordneter ZA. Ein Zertifikatsaussteller (ZA), dessen Zertifikat von einem anderen untergeordneten ZA und nicht vom Stamm-ZA unterzeichnet ist. Siehe auch Zertifikatskette, Stamm-ZA.

Unterscheidungsname (Distinguished Name, DN). Speziell formatierter Name, der den Gegenstand eines Zertifikats eindeutig identifiziert.

Urheberschaftsbeleg. Unwiderlegbarer Beweis, dass eine bestimmte Person der Urheber oder Absender einer Nachricht ist. Eine handschriftliche Unterschrift ist eine Form des Urheberschaftsbelegs. Eine andere Form ist eine digitale Signatur.

Verschlüsselung. Vorgang, bei dem Informationen unlesbar gemacht werden, um ihre Bedeutung zu verschleiern. Verschlüsselte Verbindungen zwischen Computern erschweren unbefugten Dritten das Lesen oder Entschlüsselnder Informationen, die über diese Verbindung fließen. Verschlüsselte Informationen können nur mithilfe eines geeigneten Schlüssels wieder entschlüsselt werden. Siehe auch Public-Key-Kryptografie.

Verschlüsselungszertifikat. Zertifikat, dessen öffentlicher Schlüssel einem nur für die Verschlüsselung verwendeten privaten Schlüssel entspricht. Verschlüsselungszertifikate werden nicht für Signaturoperationen verwendet. Siehe auch Doppelte Schlüsselpaare, Unterzeichnungszertifikat.

Vertrauen. Sich auf eine andere Person oder Einrichtung verlassen. Im Zusammenhang mit Public-Key-Infrastruktur (PKI)bezeichnet Vertrauen in der Regel die Beziehung zwischen dem Benutzer eines Zertifikats und dem Zertifikatsaussteller (ZA), der das Zertifikat ausgegeben hat. Wenn Sie dem Zertifikats-Manager mitteilen, dass Sie einem ZA vertrauen, akzeptiert der Zertifikats-Manager gültige Zertifikate dieses Ausstellers, sofern Sie nicht für einzelne Zertifikate andere Einstellungen vorgenommen haben. Auf der Registerkarte Aussteller im Zertifikats-Manager können Sie angegeben, welcher Art von Zertifikaten eines bestimmten Ausstellers Sie vertrauen oder nicht vertrauen.

Webseite. Ein einzelnes Dokument im World Wide Web, das mit einer eindeutigen Adresse bzw. einer URL angegeben wird. Es kann Text, Hyperlinks und Grafiken enthalten.

Website. Eine Gruppe verwandter Webseiten, die durch Hyperlinks verbunden sind und von einem Unternehmen, einer Organisation oder einer Einzelperson verwaltet werden. Eine Website kann Text, Grafiken, Audio- und Videodateien und Verknüpfungen zu anderen Websites enthalten.

World Wide Web. Auch kurz Web genannt. Ein Teil des Internets, das aus Webseiten besteht, die von Webservern gespeichert und von Clients, so genannten Webbrowsern (wie Netscape Navigator), angezeigt werden.

ZA. Siehe Zertifikatsaussteller (ZA).

ZA-Zertifikat. Zertifikat, das einen Zertifikatsaussteller identifiziert. Siehe auch Zertifikatsaussteller (ZA), Untergeordneter ZA, Stamm-ZA.

Zertifikat. Digitales Äquivalent eines Personalausweises. Ein Zertifikat gibt den Namen einer Person, Firma oder anderen Einrichtung an und bestätigt, dass ein im Zertifikat enthaltener, öffentlicher Schlüssel Eigentum dieser Einrichtung ist. Wenn Sie eine Nachricht oder andere Daten digital signieren, wird die Signatur mithilfe des privaten Schlüssels erstellt, der dem öffentlichen Schlüssel in Ihrem Zertifikat entspricht. Ein Zertifikat wird von einem Zertifikatsaussteller (ZA)ausgestellt und digital signiert. Die Gültigkeit eines Zertifikats kann durch Überprüfung der digitalen Signatur des ZA festgestellt werden. Auch bezeichnet als digitale ID, digitaler Ausweis, Public-Key-Zertifikat, X.509-Zertifikat oder Sicherheitszertifikat. Siehe auch Public-Key-Kryptograie.

Zertifikatsaussteller (ZA). Ein Dienst, der nach Überprüfung der Identität einer Person oder Einrichtung ein Zertifikat zu deren Identifikation ausstellt. Ein ZA kann Zertifikate auch erneuern oder widerrufen. Eine Liste widerrufener Zertifikate wird in regelmäßigen Abständen veröffentlicht. ZAs können unabhängige Dienstleister sein (wie die unter Client-Zertifikate aufgelisteten) oder eine Person oder Organisation, die eine Serversoftware zur Zertifikatsausstellung (z. B. Mozilla Certificate Management System) verwendet. Siehe auch Zertifikat, Zertifikatsrücknahmeliste (Certificate Revocation List, CRL).

Zertifikatsauswertung. Wenn der Zertifikats-Manager ein Zertifikat auswertet, stellt er fest, ob die digitale Signatur von einem ZA erstellt wurde, dessen ZA-Zertifikat im Zertifikats-Manager gespeichert und als vertrauenswürdig für die Ausstellung eines Zertifikats dieser Art gekennzeichnet ist. Es wird außerdem überprüft, ob das Zertifikat selbst als vertrauenswürdig gilt. Sofern das Online Certificate Status Protocol (OCSP) aktiviert wurde, führt der Zertifikats-Manager auch eine Onlineprüfung durch. Dabei wird das Zertifikat anhand einer Liste gültiger Zertifikate überprüft. Diese Liste wird unter einem bestimmten URL aufbewahrt, der direkt im Zertifikat oder in den Auswertungseinstellungen des Browsers angegeben ist. Wenn eine dieser Prüfungen fehlschlägt, markiert der Zertifikats-Manager dieses Zertifikat als unbestätigt und akzeptiert es nicht zur Identifikation.

Zertifikatserneuerung. Die Verlängerung eines Zertifikats, dessen Gültigkeit abläuft.

Zertifikatsfingerabdruck. Einmalige Nummer, die einem Zertifikat zugeordnet wird. Die Nummer ist nicht Bestandteil des Zertifikats selbst, sondern wird erzeugt durch die Anwendung mathematischer Funktionen auf den Inhalt des Zertifikats. Wenn der Inhalt des Zertifikats auch nur um ein einziges Zeichen geändert wurde, ergibt die Funktion eine andere Zahl. Daher kann mithilfe von Zertifikatsfingerabdrücken festgestellt werden, ob ein Zertifikat manipuliert wurde.

Zertifikatskette. Hierarchische Abfolge von Zertifikaten, die von aufeinanderfolgenden Zertifikatsausstellern (ZAs) signiert sind. Ein ZA-Zertifikat identifiziert einen Zertifikatsaussteller (ZA) und wird verwendet, um die von diesem Aussteller ausgegebenen Zertifikate zu signieren. Ein ZA-Zertifikat kann wiederum vom ZA-Zertifikat eines übergeordneten Ausstellers bis hin zum Stamm-ZA signiert sein.

Zertifikats-Manager Der Teil des Browsers, mit dem Sie Zertifikate anzeigen und verwalten können. So öffnen Sie das Hauptfenster des Zertifikats-Managers: Klicken Sie im Menü Bearbeiten auf Einstellungen, klicken Sie auf Privatsphäre und Sicherheit und dann auf Zertifikate verwalten.

Zertifikatsrücknahmeliste (Certificate Revocation List, CRL). Liste mit allen widerrufenen Zertifikaten, die regelmäßig von einem Zertifikatsaussteller (ZA) erstellt und signiert wird. Sie können die jeweils aktuelle CRL in Ihren Browser oder auf einen Server herunterladen, um Zertifikate anhand dieser Liste auf ihre Gültigkeit zu überprüfen.

Zertifikatssicherungskennwort. Passwort, das eine Sicherheitskopie eines Zertifikats schützt. Der Zertifikats-Manager fordert Sie bei der Sicherung von Zertifikaten zur Eingabe eines Passworts auf, das später abgefragt wird, wenn Sie die Zertifikate wiederherzustellen versuchen.
