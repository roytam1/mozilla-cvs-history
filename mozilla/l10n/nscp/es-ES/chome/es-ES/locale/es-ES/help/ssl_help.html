Versiones de protocolos SSL

El protocolo SSL (Capa de seguridad de circuitos) define las reglas que rigen la autenticación mutua entre un sitio web y el software del navegador y el cifrado de la información que fluye entre ellos. El protocolo TLS (Seguridad en la capa de transporte) es una norma de IETF basada en SSL. Se puede entender TLS 1.0 como SSL 3.1.

Se recomienda que active estas tres casillas de verificación para asegurarse de que todos los servidores web, tanto los nuevos como los más antiguos, son compatibles con el navegador:

  • Habilitar SSL versión 2: permite a los servidores web más antiguos funcionar con el navegador.
  • Habilitar SSL versión 3: permite a los servidores web más nuevos funcionar con el navegador.
  • Habilitar TLS: permite aprovechar TLS a los servidores web compatibles.

Para activar o desactivar cifrados de forma independiente, haga clic en el botón Editar cifrados.

  • Editar cifrados: no modifique los cifrados separadamente a no ser que sepa exactamente lo que está haciendo y haya consultado los cambios con el administrador del sistema. Si desea más información, consulte Editar cifrados.

Aviso importante sobre TLS: algunos servidores que no implantan SSL correctamente no pueden negociar la aprobación de SSL con el software del cliente (como el navegador) que admite TLS. Tales servidores se conocen como "intolerantes hacia TLS".

Cuando está seleccionada la opción Habilitar TLS en el panel de preferencias de SSL, el navegador intenta utilizar el protocolo TLS al realizar conexiones seguras con un servidor. Si esa conexión falla debido a que el servidor es intolerante hacia TLS, el navegador volverá a utilizar SSL 3.0.

 

Advertencias de SSL

Es fácil distinguir cuándo el sitio web que está viendo utiliza una conexión cifrada. Si la conexión está cifrada, el icono del candado de la esquina inferior derecha de la ventana de Navigator aparece cerrado. Si la conexión no está cifrada, el candado está abierto.

Si desea consultar más advertencias, maque una o varias de las casillas de advertencias del panel de preferencias de SSL. Hay gente a la que estas advertencias le resultan molestas.

Para activar cualquiera de las advertencias de Navigator, marque la casilla de verificación correspondiente:

  • Carga de una página que admite el cifrado. Seleccione esta advertencia si desea recibir un aviso siempre que cargue una página que admite cifrado.
  • Carga de una página que utiliza cifrado de nivel bajo. Seleccione esta advertencia si desea recibir un aviso siempre que cargue una página que admite cifrado. (El cifrado de nivel bajo es el cifrado más débil disponible, sólo utiliza claves de 40 bits).
  • Salida de una página que admite cifrado. Seleccione esta advertencia si desea recibir un aviso siempre que salga de una página que admite o no cifrado.
  • Envío de datos de un formulario desde una página no cifrada a una página no cifrada. Seleccione esta advertencia si desea recibir un aviso siempre que envíe datos con una conexión no cifrada. Si envía información no cifrada por Internet, puede ser interceptada fácilmente por otras personas.
  • Vista de una página con una mezcla de cifrado/no cifrado. Seleccione esta advertencia si desea recibir una alerta siempre que vea una página con cualquier información que no esté cifrada.

Si desea acceder a las definiciones cortas, haga clic en autenticación, cifrado o certificado.

Editar cifrados

En esta sección se explica el uso del cuadro de diálogo Editar cifrados. Si aún no ha llegado al panel, siga los pasos que se indican a continuación:

  1. Abra el menú Editar y seleccione Preferencias.
  2. En la categoría Privacidad y seguridad, haga clic en SSL. (Si no aparece ninguna subcategoría, haga doble clic en Privacidad y seguridad para expandir la lista.)
  3. Haga clic en el botón Editar cifrados.

El cuadro de diálogo Editar cifrados permite activar o desactivar cifrados, o algoritmos criptográficos. Consulte con el administrador del sistema antes de cambiar estos valores.

Cada una de las casillas de verificación corresponde a un conjunto cifrado o a un conjunto de algoritmos criptográficos.

Si desea más información sobre cifrado y criptografía, consulte en Internet los siguientes documentos:

  • Introduction to Public-Key Cryptography (Introducción a la criptografía de clave pública).
  • Introduction to SSL (Introducción a SSL)
  • Encryption Technologies Available in NSS 3.4 (Tecnologías de cifrado disponibles en NSS 3.4).
