身分証明書

証明書マネージャ の［身分証明書］タブには、ユーザの身元証明として保管されている証明書が表示されます。 証明書は、それらを発行した組織ごとに表示されます。

  • 組織名の下に証明書名が表示されていない場合は、組織名をダブルクリックします。
  • 証明書を選択するには、その証明書の名前をクリックします。
  • 複数の証明書を選択するには、Ctrl キーを押しながら証明書名をクリックします。

以下の操作を行うには、1 つまたは複数の証明書を選択して、以下のボタンのいずれかをクリックします。

  • 表示：選択した証明書の詳細情報を表示します。
  • バックアップ： 選択した証明書の保存プロセスを開始します。表示されたウィンドウで、証明書のバックアップを保護するためのパスワードを選択します。続いて、証明書のバックアップを指定のディレクトリに保存します。
  • 削除：選択した証明書を削除します。

以下の操作では、証明書を選択する必要がありません。

  • インポート：以前にバックアップした 1 つ以上の証明書を含むファイルをインポートします。 ［インポート］をクリックすると、まず証明書のバックアップ ファイルを指定するよう指示されます。証明書のバックアップ ファイル名の末尾には、通常、.p12 という拡張子が付いています（例： MyCert.p12など）。 インポートするファイルを選択すると、その証明書のバックアップ時に設定したパスワードを入力するよう指示されます。
  • すべてバックアップ：ソフトウェア セキュリティ デバイス に保管されているすべての証明書を保存するプロセスが開始します。

    注記： スマート カードに保存されている証明書をバックアップすることはできません。 証明書を選択して［バックアップ］や［すべてバックアップ］をクリックしても、作成されるバックアップ ファイルには、スマート カードやその他の外部セキュリティ デバイスに保存されている証明書は含まれません。 バックアップできるのは、内蔵の ソフトウェア セキュリティ デバイス に保存されている証明書だけです。

 

証明書バックアップ パスワードの選択

証明書バックアップ パスワードは、証明書マネージャの［ 身分証明書 ］タブからバックアップを作成する際に、それらの証明書を保護するためのものです。

証明書をバックアップするときにはパスワードを設定するよう求められ、バックアップした証明書をインポートするときにはそのパスワードを入力するよう求められます。

  • 証明書バックアップ パスワード：このフィールドにバックアップ用のパスワードを入力します。
  • 証明書バックアップ パスワード（もう一度）：バックアップ パスワードをもう一度入力します。 最初と同じように入力しないと、［OK］ボタンが使えるようになりません。その場合は、新しいパスワードをもう一度入力してみてください。

バックアップした証明書を含むファイルを他人が取得して証明書のインポートに成功した場合、その人は本来のユーザになりすましてメッセージを送信したり Web サイトにアクセスすることができます。 これは、たとえば、重要な電子メール メッセージにデジタル署名したり、インターネットで銀行口座や投資口座を管理しているユーザにとって、非常に危険なことです。

そのため、他人が推測できないような証明書バックアップ パスワードを選択することが重要になります。パスワード品質メーターは、入力したパスワードの長さ、大文字、小文字、数値、記号の使い方などを基にして、パスワードの品質を大まかに知らせてくれます。 ただし、パスワードが絶対に解読されないという保証はありません。

詳しくは、オンライン マニュアルの「Choosing a Good Password（最適なパスワードを選択する）」を参照してください。

パスワードを安全な場所に記録しておくことも大切です。他人が簡単にアクセスできる場所には保管しないでください。 パスワードを忘れると、証明書のバックアップをインポートできなくなります。

 

身分証明書の削除

証明書マネージャの［身分証明書］で期限切れになった自分の証明書を削除する前に、対応する秘密キーで暗号化された古い電子メールを後で読む必要がないことを確認してください。

 

他の人びとの証明書

証明書マネージャの［他の人びと］タブには、他の人びとを識別するために保管されている電子メール証明書が表示されます。

デジタル署名の付いた電子メール メッセージを受け取ると、証明書マネージャがその証明書を自動的にインポートします。 これらの証明書を使って、それらの人びとに暗号化メッセージを送信することができます。

他の人びとの証明書は、それらを発行した組織ごとに表示されます。

  • 組織名の下に証明書名が表示されていない場合は、組織名をダブルクリックします。
  • 証明書を選択するには、その証明書の名前をクリックします。
  • 複数の証明書を選択するには、Ctrl キーを押したまま証明書名をクリックします。

以下の操作を行うには、1 つ以上の証明書を選択し、以下のボタンのいずれかをクリックします。

  • 表示：選択した証明書の詳細情報を表示します。
  • 削除：選択した証明書を削除します。

 

電子メール証明書の削除

証明書マネージャの［他の人びと］タブで他人の証明書を削除する前に、その人に暗号化されたメールを今後送信する必要がないかどうか、また、その人から送られてきたメッセージのデジタル署名を検証する必要がないかどうかを確認してください。

 

Web サイトの証明書

証明書マネージャの［Web サイト］タブには、Web サイトを識別するために保管されている証明書が表示されます。

Web サイト証明書は、それらを発行した組織ごとに表示されます。

  • 組織名の下に証明書名が表示されていない場合は、組織名をダブルクリックします。
  • 証明書を選択するには、その証明書の名前をクリックします。
  • 複数の証明書を選択するには、Ctrl キーを押したまま証明書名をクリックします。

以下の操作を実行するには、1 つ以上の証明書を選択し、以下のボタンのいずれかをクリックします。

  • 表示：選択した証明書の詳細情報を表示します。
  • 編集：選択した証明書に関連付けられた信頼設定を表示または変更できます。これらの設定を使って、ID を確認する上で Web サイト証明書を信頼するかどうかを指定します。
  • 削除：選択した証明書を削除します。

 

Web サイト証明書の信頼設定を編集

証明書マネージャの［Web サイト］タブで Web サイト証明書を選択して［編集］をクリックすると、［Web サイト証明書の信頼設定を編集］ウィンドウが表示されます。 このウィンドウでは、Web サイトを識別し、かつその Web サイトとの間で暗号化された接続をセットアップするために選択した証明書を信頼するかどうかを指定します。

このダイアログボックスには、以下の要素があります。

  • 証明書「証明書名」の発行者： この証明書を発行した認証局（CA）に関する情報を表示します。
  • 証明書の信頼設定を編集：
    • この証明書の認証を信頼する：このオプションを選択すると、証明書マネージャは、この Web サイトの識別または暗号化接続の設定という目的において、今後この証明書を信頼します。 このオプションを選択した後は、その Web サイトにアクセスしようとしても、警告が表示されることはほとんどありません。
    • この証明書の認証を信頼しない：このオプションを選択すると、証明書マネージャは、この Web サイトの識別または暗号化接続の設定という目的において、今後この証明書を信頼しなくなります。 このオプションを選択した後は、その Web サイトにアクセスしようとすると、1 つまたは複数の警告が表示されます。
  • 認証局の信頼設定を編集： Web サイト証明書を発行した認証局（CA）の信頼設定を指定できます。 この設定によって、その認証局によって発行される様々な種類の証明書を信頼するかどうかを指定します。 たとえば、その認証局によって発行されるすべての Web サイト証明書を信頼するように選択できます。

設定内容を確認したら［OK］をクリックします。

 

Web サイト証明書の削除

証明書マネージャの［Web サイト］タブで Web サイト証明書を削除する前に、今後その証明書を使って Web サイトを識別したり暗号化された接続をセットアップしたりする必要がないことを確認してください。

 

認証局

証明書マネージャの［認証局］タブには、認証局（CA）を識別するために保管されている証明書が表示されます。

CA 証明書は、それらを発行した組織ごとに表示されます。

  • 組織名の下に CA 証明書名が表示されていない場合は、組織名をダブルクリックします。
  • CA 証明書を選択するには、その証明書の名前をクリックします。
  • 複数の CA 証明書を選択するには、Ctrl キーを押したまま証明書名をクリックします。

以下の操作を行うには、目的の証明書を選択し、いずれかのボタンをクリックします。

  • 表示：選択した証明書の詳細情報を表示します。
  • 編集：選択した証明書に関連付けられている設定を表示または変更できます。これらの設定を使って、認証局が発行したどの証明書を信頼するかを指定できます。
  • 削除：選択した証明書を削除します。

認証局の証明書チェーン全体が信頼できるようにするには、ルート CA 証明書を編集するだけで済みます。

チェーンをインポートするには、CA が提供する Web ページでリンクをクリックします。 次に［認証局］タブを使ってルート証明書を検索し、その信頼設定を編集します。

ルート CA と中間 CA がすべて同じ組織名の下に表示されます。 発行者自身として表示されているのがルート証明書です。

中間 CA をダウンロードした場合： お使いのブラウザによって信頼されているルート証明書にチェーンされた中間 CA 証明書をダウンロードする場合、その証明書を何の目的で信頼するのかを指定する必要はありません。 中間証明書には、自動的にルート証明書の信頼設定が適用されます。

 

認証局証明書の信頼設定を編集

証明書マネージャの［認証局］タブで CA 証明書を選択して［編集］をクリックすると、［認証局の証明書の信頼設定を編集］ウィンドウが表示されます。このウィンドウでは、この認証局の証明書によって信頼する証明書の種類を指定します。どのチェックボックスも選択しなかった場合、証明書マネージャは、この認証局が発行する証明書を一切信頼しません。

以下のオプションがあります。

  • この証明書によって、Web サイトを識別できます：証明書マネージャは、Web サイトの識別および Web サイト接続の暗号化という目的において、認証局が発行した証明書を信頼します。このチェックボックスをオンにしなかった場合、証明書マネージャは、この認証局が発行した Web サイト証明書を信頼しません。
  • この証明書によって、メール ユーザを識別できます：証明書マネージャは、Web サイトの識別および電子メールの暗号化という目的において、認証局が発行した証明書を信頼します。このチェックボックスをオンにしなかった場合、証明書マネージャは、この認証局が発行した電子メール証明書を信頼しません。
  • この証明書によって、ソフトウェアの製造元を識別できます：証明書マネージャは、ソフトウェア メーカーの識別という目的において、認証局が発行した証明書を信頼します。このチェックボックスをオンにしなかった場合、証明書マネージャは、この認証局が発行した証明書を信頼しません。

設定内容を確認したら［OK］をクリックします。

 

認証局証明書の削除

証明書マネージャの［認証局］タブで CA 証明書を削除する前に、この認証局によって発行される証明書を今後再び検証する必要がないかどうかを確認してください。認証局について唯一の有効な証明書を削除すると、証明書マネージャは、この認証局が発行する証明書を一切信頼しなくなります。

 

デバイス マネージャ

このセクションでは、［デバイス マネージャ］ウィンドウで使用できるオプションについて説明します。デバイス マネージャに関する詳細情報や使用手順については、｢スマート カードやその他のセキュリティ デバイスを管理する｣を参照してください。

［デバイス マネージャ］ウィンドウを表示するには、以下の手順で操作します。

  1. ［編集］メニューから［設定］を選択します。
  2. ［プライバシーとセキュリティ］カテゴリの［証明書］をクリックします（サブカテゴリが表示されていない場合は、［プライバシーとセキュリティ］をダブルクリックしてリストを展開表示します）。
  3. ［証明書］パネルで、［セキュリティ デバイスの管理］をクリックします。

デバイス マネージャでは、使用可能な PKCS #11 モジュールが示され、その下に各モジュールによって管理されているセキュリティ デバイスが表示されます。

モジュールまたはデバイスを選択すると、その項目に関する情報がウィンドウの中央に表示され、右側にあるボタンのいくつかが使用できるようになります。 一般に、モジュールまたはデバイスの名前を選択し、適切なボタンをクリックすると、操作が実行されます。

  • ログイン：選択したセキュリティ デバイスにログインします。 ログイン後にそのデバイスに対するマスター パスワードの入力を求められる頻度は、［マスター パスワードの時間切れ］設定によって異なります。
  • ログアウト：選択したセキュリティ デバイスからログアウトします。 いったんログアウトすると、再びログインするまでそのデバイスとそれに含まれる証明書は使用できません。
  • パスワードの変更：選択したセキュリティ デバイス用のマスター パスワードを変更できます。
  • 読み込み：新しい PKCS #11 モジュールの名前とその場所を指定するためのダイアログ ボックスを表示します。新しいモジュールを追加する前に、まずモジュールのソフトウェアをコンピュータにインストールし、必要に応じて、関連するハードウェア デバイスを接続しておく必要があります。ベンダーからの指示に従ってください。
  • 読み込み解除：選択したモジュールの読み込みを解除します。 モジュールの読み込みを解除すると、モジュールとそのセキュリティ デバイスはブラウザで使用できなくなります。
  • FIPS を使用する：FIPS モードのオンとオフを切り替えます。詳しくは、「FIPS モードを使用する」を参照してください。
