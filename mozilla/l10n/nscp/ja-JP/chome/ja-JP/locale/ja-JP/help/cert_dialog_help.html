セキュリティ デバイスの選択

セキュリティ デバイス（トークンとも呼ばれる）は、暗号化や復号化のような暗号処理サービスを行うハードウェアまたはソフトウェアから構成されるデバイスで、証明書とキーを保管します。証明書やキーをインポートしたり、新しい証明書用のキーを生成したりするなど、暗号化処理を実行する際に使用するセキュリティ デバイスを決定するときには、［セキュリティ デバイスの選択］ウィンドウが表示されます。このウィンドウで、ユーザのコンピュータから検出された複数のセキュリティ デバイスの中から目的のものを選択します。

セキュリティ デバイスの一例が、スマート カードです。たとえば、コンピュータに接続されたスマート カード リーダーにスマート カードが挿入されている場合、そのスマート カードの名前がドロップダウン メニューに表示されます。この場合、メニューからスマート カードの名前を選択して、それを使用することを証明書マネージャに知らせる必要があります。

また、証明書マネージャには独自の標準セキュリティ デバイスも備わっています。このデバイスは、他のデバイスが使用可能であるかどうかに関わらず、常に使用できます。

 

暗号化キーのコピー

署名証明書と暗号化電子メール証明書を別々に発行する認証局（CA）では通常、証明書の加入プロセス時に、ユーザの秘密暗号化キーのバックアップ コピーが作成されます。

［暗号化キーのコピー］ダイアログ ボックスでは、バックアップ コピーの作成を許可、あるいは証明書の要求をキャンセルすることができます。 暗号化キーのバックアップ コピーを保存した認証局では、対応する公開キーを使って暗号化されたメッセージをユーザが受け取った場合、それを解読することが可能となります。

［暗号化キーのコピー］ダイアログ ボックスでは以下の操作を実行できます。

  • 証明書を表示： バックアップ コピーを要求している認証局を識別する証明書を参照するには、［証明書を表示］をクリックします。

  • OK： CA 証明書によって識別される認証局を信頼し、受信した暗号化メッセージの解読を許可する場合は［OK］をクリックします。

    バックアップ コピーを要求している認証局が信頼できるかどうか確かでない場合は、システム管理者に問い合わせてください。

  • キャンセル：バックアップ コピーを要求している認証局を信頼しない場合は、そこから証明書を要求しないでください。 ［キャンセル］をクリックして、バックアップ手続きと証明書要求手続きを中止します。

認証局が暗号化キーのバックアップ コピーを作成すると、後でパスワードを忘れたり、暗号化キーのコピーを紛失した場合でも、バックアップ キーを使って暗号化されたメールにアクセスすることができます。 暗号化キーのバックアップ コピーがないと、パスワードやキーをなくした場合、そのキーを使って暗号化された電子メール メッセージを読むことはできなくなります。

 

証明書のバックアップ

証明書を受信したら、その証明書と秘密キーのバックアップをとり、そのコピーを安全な場所に保管してください。たとえば、コピーをフロッピー ディスクに格納し、他の貴重品と同様に鍵のかかる場所に保管するとよいでしょう。こうすれば、ハード ディスクやファイルが損傷した場合でも、簡単に証明書を復元することができます。

用途によっては、証明書や関連する秘密キーの紛失は不便だけでなく深刻な問題をもたらすこともあります。例として、次の状況が考えられます。

  • 重要な Web サイトに対してユーザを識別する証明書を紛失すると、新しい証明書を取得するまで、その Web サイトにアクセスできなくなります。
  • 電子メールを暗号化する証明書を紛失すると、送信および受信した暗号化メールが読めなくなくなります。この場合、証明書に関連付けられた秘密キーのバックアップを取得できないと、そのキーで暗号化されたメッセージは二度と読むことができません。

他の貴重なデータと同様に、証明書のバックアップをとり、トラブルや不要な出費を避けるようにしてください。忘れないうちに、今すぐバックアップを実行しましょう。

 

ユーザ ID の要求

Web サイトの中には証明書によってユーザ識別を行う Web サイトもあります。これは、名前やパスワードよりも証明書の方がより信頼性の高い ID と考えられるためです。インターネット上でのこのユーザ識別方法は、クライアント認証とも呼ばれます。

ただし、Web サイトに対してユーザ ID を証明する証明書が複数個ある場合は、証明書マネージャにより、［ユーザ ID の要求］ダイアログ ボックスに 2 種類の情報が表示されます。

証明書を使用して自分の身元を証明するようにサイトから要求されました： ダイアログ ボックスのこのセクションには、以下の情報が表示されます。

  • ホスト名：身元証明を要求しているサーバ名で、URLの一部として使用されます。たとえば、Netscape の Web サイトのホスト名はhome.netscape.comです。
  • 組織： その Web サイトを運営している組織名。
  • 発行： 証明書を発行した認証局（CA）名。

ID として提示する証明書を選択してください：このセクションのドロップダウン メニューには、Web サイトに対してユーザが身元証明として使用できる証明書が表示されます。アクセスしたい Web サイトによって最も認識される可能性の高い証明書を選択してください。

参考のため、選択された証明書に関する以下の詳細情報が表示されます。

  • 発行対象： 証明書によって身元が証明されるユーザに関する情報（たとえば氏名や電子メール アドレスなど）と、証明書のシリアル番号、有効期限などが表示されます。
  • 発行者： 証明書を発行した認証局の概略（たとえば認証局名や場所、都道府県名など）が表示されます。

 

新しい認証局

証明書マネージャがファイルに格納している証明書の中には、コンピュータまたはスマート カードなどの外付けセキュリティ デバイスのどこに格納されているかに関わらず、認証局(CA)を識別するものが含まれています。ファイルに保存されているその他の証明書を認識する上で、証明書マネージャは、それらの証明書を発行した CA または認定発行者の証明書を必要とします。

特定の CA を信頼することにした場合、証明書マネージャは CA の証明書をダウンロードし、その CA が発行する証明書を認識するようになります。

新しい CA 証明書をダウンロードする前に、信頼する証明書の目的があれば、それを証明書マネージャで指定できます。次のうち、いずれかのオプションを選択できます。

  • Web サイトを識別するためにこの認証局を信頼する：金銭のやり取りを扱う Web サイトの証明書は非常に重要です。不適切な識別や誤った識別は、問題を起こす危険性があります。
  • 電子メール ユーザを識別するためにこの CA を信頼する：電子メール ユーザに暗号化された機密情報を送信する場合、あるいは何らかの理由で電子メール ユーザの正確な ID が重要である場合は、このオプションを選択する前に、認証局が証明書所有者の ID を識別する手順や、それが目的にかなったものかどうかを検討する必要があります。
  • ソフトウェア デベロッパを識別するためにこの認証局を信頼する：このオプションを選択すると、ファイル変更などの特別な権限で、ユーザのコンピュータにアクセスする Java アプレットや JavaScript の開発者を識別する証明書を、認証局が発行できるようになります。このような特権的アクセスが誤用されると、ハード ディスク内に格納されたデータが破壊される危険性があります。この目的に関して認証局を信頼してよいかどうかを確信できない場合は、このオプションの選択について慎重に判断する必要があります。

新しい認証局を信頼する前に、誰がその認証局を運営しているかを確認してください。発行される証明書にとって認証局の方針や手順が適切かどうか、たとえば、認証局が金銭のやり取りを行う Web サイトの証明書を発行する場合は、その認証局の保証レベルが満足すべきものであることを確認してください。

  • 表示： ダウンロードする CA 証明書を表示するには、このボタンをクリックします。 この証明書のダウンロードを取り止めるには、［キャンセル］をクリックします。

サーバ証明書の有効期限が切れました

クレジット カード、運転免許証、およびその他の多くの身分証明書と同様、証明書は指定された期間だけ有効です。有効期限が切れた場合は、新しい証明書を取得する必要があります。

サーバ証明書の有効期限が切れている Web サイトにアクセスしようとすると、証明書マネージャにより警告が表示されます。 まず、お使いのコンピュータの日時が正確かどうかをチェックしてください。コンピュータの時刻がこの期限以降の日付になっている場合、証明書マネージャは、Web サイトの証明書を期限切れとして処理します。

コンピュータで日時が正しく設定されている場合は、そのサイトを信頼するかどうかの判断が必要となります。 信頼するかどうかの判断は、そのサイトについてどの程度分っているか、また、そのサイトで何をするかによります。ほとんどの商用サイトは、証明書の有効期限が切れる前に証明書の更新を行っています。

［サーバ証明書の有効期限が切れました］ダイアログ ボックスでは、以下の操作を行うことができます。

  • 証明書を表示：有効期間など、証明書に関する情報をチェックするには、［証明書を表示］をクリックします。

  • OK：証明書の期限切れが不注意によるものであると判断した場合は、［OK］をクリックして今回のセッションで証明書を受け入れ、Web サイトの管理者にこの問題を通知します。

    そのサイトにアクセスして操作を行う際には、特に警戒が必要です。

  • キャンセル： その Web サイトに重大な問題があると考えられる場合は、［キャンセル］をクリックしてアクセスを中止します（証明マネージャはそのサイトに接続しません）。

 

サーバ証明書はまだ有効ではありません

クレジット カード、運転免許証、およびその他の多くの身分証明書と同様、証明書は指定された期間だけ有効です。

サーバ証明書がまだ有効になっていない Web サイトにアクセスしようとすると、証明書マネージャにより警告が表示されます。まず、コンピュータの日付が正確かどうかをチェックしてください。 コンピュータの日付および時刻表示が間違っている場合、証明書マネージャは、その時間設定に従って、サーバ証明書がまだ有効でないと判断する可能性があります。

コンピュータで日時が正しく設定されている場合は、そのサイトを信頼するかどうかの判断が必要となります。 信頼するかどうかの判断は、そのサイトについてどの程度分っているか、また、そのサイトで何をするかによります。ほとんどの商用サイトは、証明書の使用を開始する前に、証明書が有効期間内にあることを確認しています。

［サーバ証明書はまだ有効ではありません］ダイアログ ボックスでは、以下の操作を行うことができます。

  • 証明書を表示：有効期間など、証明書に関する情報をチェックするには、［証明書を表示］をクリックします。

  • OK：問題が不注意によるものであると判断した場合は、［OK］をクリックして今回のセッションで証明書を受け入れ、Web サイトの管理者にこの問題を通知します。

    そのサイトにアクセスして操作を行う際には、特に警戒が必要です。

  • キャンセル： その Web サイトに重大な問題があると考えられる場合は、［キャンセル］をクリックしてアクセスを中止します（証明マネージャはそのサイトに接続しません）。

 

ドメイン名の不一致

サーバ証明書では、サイトのホスト名を使ってサーバ名が指定されいます。 たとえば、Netscape の Web サイトのドメイン名は home.netscape.com です。 サーバ証明書内のドメイン名が Web サイトの実際のドメイン名と一致しないということは、ユーザと Web サイト間の通信を、第三者が傍受しようとしている可能性があることを意味します。

サイトを信頼すべきかどうかは、サイトで何をしようとしているのか、サイトについてどのような知識があるかによって決まります。ほとんどの商用サイトの場合は、Web サイト証明書に記載されているホスト名が Web サイトの実際のホスト名と一致しています。

［ドメイン名の不一致］ダイアログ ボックスでは、以下の操作を行うことができます。

  • 証明書を表示：証明書の情報を検証するには、［証明書を表示］をクリックします。

  • OK：問題が不注意によるものであると判断した場合は、［OK］をクリックして今回のセッションで証明書を受け入れ、Web サイトの管理者にこの問題を通知します。

    そのサイトにアクセスして操作を行う際には細心の注意を払い、そのサイトで取得した情報を慎重に取り扱うようにしてください。

  • キャンセル： その Web サイトに重大な問題があると考えられる場合は、［キャンセル］をクリックしてアクセスを中止します（証明マネージャはそのサイトに接続しません）。

現在のセッションで証明書を受け入れる場合は、Web サイトでの操作を慎重に行い、そのサイトで取得した情報を慎重に取り扱うようにしてください。
