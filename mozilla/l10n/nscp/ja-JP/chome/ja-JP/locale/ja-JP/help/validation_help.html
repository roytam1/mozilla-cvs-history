プライバシー とセキュリティの設定 - 確認

このセクションでは、［確認］設定パネルの使い方について説明します。この設定パネルを表示するには、以下の手順で操作します。

  1. ［編集］メニューから［設定］を選択します。
  2. ［プライバシーとセキュリティ］カテゴリの［確認］をクリックします（サブカテゴリが表示されていない場合は、［プライバシーとセキュリティ］カテゴリをダブルクリックしてリストを展開表示します）。

証明書の確認については、「確認機能を制御する」を参照してください。

 

CRL

証明書破棄リスト(CRL)とは、認証局（CA）によって作成および署名された、破棄済みの証明書のリストです。CRL をブラウザにダウンロードしておけば、証明書を認証目的で使用する前に、まだ証明書が有効かどうかを確認することができます。

［CRL の管理］をクリックすると、証明書マネージャで使用できる CRL のリストが表示されます。

CRL の管理について詳しくは、「CRL を管理する」を参照してください。

 

OCSP

OCSP（Online Certificate Status Protocol）は、証明書が表示または使用されるたびに、証明書マネージャが証明書の有効性をオンラインでチェックできるようにするためのものです。このプロセスでは、指定された Web サイトで管理されている証明書破棄リスト（CRL）に対して、証明書がチェックされます。OCSP が機能するためには、コンピュータがオンライン状態でなければなりません。

証明書マネージャが OCSP を使用する方法を指定するには、［確認］の OCSP 設定から以下のいずれかの設定を選択します。

  • 証明書の確認に OCSP を使用しない：証明書マネージャが証明書を確認するたびにオンライン状態チェックを実行したくない場合は、このオプションをオンにします。その代わり、証明書マネージャは、証明書の確認を実行するたびに、証明書の有効期間を確認し、また、認証局によって正しく署名されていることだけを確認します。この場合の認証局とは、（証明書マネージャのメイン ウィンドウにある）［認証局］タブに一覧されており、かつその種の証明書の発行について信頼できるとマークされた認証局証明書を持つ認証局のことです。
  • OCSP を使用して OCSP サービス URL に指定されている証明書だけを確認する：証明書マネージャは、証明書を確認するたびにオンライン ステータス チェックを実行します（そのようなチェック目的の URL が証明書に含まれている場合）。証明書に URL が含まれている場合、証明書マネージャは、その場所に証明書が有効として一覧されていることを確認し、有効期間と信頼設定についてのチェックを行います。
  • OSCP を使用してここに指定されている URL と署名者を使用しているすべての証明書を確認する：証明書マネージャが証明書を確認するたびにオンライン ステータス チェックを実行したい場合は、この設定を選択します。その場合、OCSP 応答の署名者を特定する［応答署名者］ポップアップ メニューから証明書を選択する必要があります。この設定で証明書マネージャが認める証明書は、OCSP 応答によって確認され、応答署名者証明書で（またはそれに関連した証明書を使って）署名されたものだけです。

    ポップアップ メニューから応答署名者証明書を選択すると、証明書マネージャは、その署名者の［サービス URL］にサービス URL（ある場合）を自動的に入力します。サービス URL が自動的に入力されない場合は、ユーザ自身が指定しなければなりません。 その場合は、システム管理者に問い合わせてください。

CRL の管理

このセクションでは、［CRL の管理］ダイアログ ボックスの使い方について説明します。 このダイアログ ボックスを表示するには、以下の手順で操作します。

  1. ［編集］メニューから［設定］を選択します。
  2. ［プライバシーとセキュリティ］カテゴリの［確認］をクリックします （サブカテゴリが表示されていない場合は、［プライバシーとセキュリティ］カテゴリをダブルクリックしてリストを展開表示します）。
  3. ［CRL の管理］をクリックします。

このダイアログ ボックスには、お使いのブラウザ用にダウンロードされた CRL のリストが表示されます。 通常、URL をクリックすると CRL がダウンロードされます。 CRL について詳しくは、「CRL を管理する」を参照してください。

CRL を選択するには、それをクリックします。 以下のいずれかの操作を行うことができます。

  • 削除：ハード ディスクから CRL を永久に削除する場合に選択します。 ただし、CRL が証明書の確認に不要になった場合以外は削除しないでください。 必要かどうかが確かでない場合は、システム管理者に問い合せてください。
  • 設定：［CRL 自動更新の設定］ダイアログ ボックスが開きます。このダイアログ ボックスでは、選択した CRL の自動更新機能を有効にしたり、自動更新の頻度を指定することができます。
  • 更新：選択した CRL がすぐに更新されます（更新可能な場合）。

［CRL の管理］ダイアログ ボックスには、各 CRL に関する以下の情報が表示されます。

  • 組織（O）：CRL を発行した組織名です。
  • 組織ユニット（OU）：CRL を発行した組織ユニット名です（特定の証明書に対するルート CA など）。
  • 最終更新：ブラウザにコピーされたこの CRL が最後に更新された日付です。
  • 次の更新：この CRL の更新版が CRL の発行元から発行される日付です。
  • 自動更新：この CRL の自動更新が有効になっているかどうかが示されます。 自動更新の設定を表示するには、CRL を選択してから［設定］をクリックします。
  • 自動更新のステータス：
    • 自動更新が有効になっていない場合や、有効になっていても次に予定されている更新がまだ行われていない場合、このフィールドは空白です。
    • （自動更新が 1 回以上行われた場合）最後の自動更新に失敗した場合は［失敗］が、成功した場合は［OK］が表示されます。

CRL インポート ステータス

このセクションでは、［CRL インポート ステータス］ダイアログ ボックスの使い方について説明します。このダイアログ ボックスは、初めて CRL をインポートするとき、または手動で CRL の更新を行うときに表示されます。

このダイアログ ボックスには、以下の情報が表示されます。

  • CRL のインポートまたは更新が成功したかどうか
  • CRL を発行した組織
  • この CRL の次の更新版が発行される日付
  • この CRL に対して自動更新機能が有効になっているかどうか

自動更新機能が有効になっていない場合は、このダイアログ ボックスから有効にすることができます。

  • はい：この CRL の自動更新機能を有効にします。 このボタンをクリックすると、［CRL 自動更新の設定］ダイアログ ボックスが表示されます。 次のセクションでは、これらの設定について説明します。
  • いいえ： 自動更新機能を無効のままにしておきます。

CRL 自動更新の設定

このセクションでは、［CRL 自動更新の設定］ダイアログ ボックスの使い方について説明します。 このダイアログ ボックスを表示するには、以下の手順で操作します。

  1. ［編集］メニューから［設定］を選択します。
  2. ［プライバシーとセキュリティ］カテゴリの［確認］をクリックします（サブカテゴリが表示されていない場合は、［プライバシーとセキュリティ］カテゴリをダブルクリックしてリストを展開表示します）。
  3. ［CRL の管理］をクリックして、自動更新設定を表示または変更する CRL を選択します。
  4. CRL を選択したら、［設定］をクリックします。

このダイアログ ボックスには、以下のオプションと情報が表示されます。

  • この CRL の自動更新を有効にする： ここで設定するスケジュールに従って、選択した CRL を自動更新します （CRL の次の更新日が指定されていない場合は、このオプションを選択できません）。

    自動更新を有効にする場合は、以下のラジオ ボタンのいずれかを選択する必要があります。

    • 次の更新日の X 日前に更新： CRL の発行元が新しいバージョンの CRL を発行する頻度に合わせて更新頻度を設定します。
    • 更新間隔： X 日： CRL の次の更新日に関係なく更新頻度を指定できます。
  • CRL のインポート元：ブラウザが最初の CRL をインポートしたときの URL が表示されます。 この設定は変更できません。 別の場所を指定するには、CRL を削除して新しい場所からインポートし直して下さい。
  • 以前に連続して発生した更新エラー：この CRL を更新しようとして連続的に発生したエラーの回数が表示されます。回数には最後に発生したエラーも含まれます。
    • 最後に試みた更新が成功した場合は、それ以前にエラーが発生していても［なし］と表示されます。
    • 最後に試みた更新が失敗した場合は、連続して発生したエラーの回数と最後に試みた更新に対するエラー メッセージが表示されます。

設定内容を確認したら［OK］をクリックします。
