この用語集は参考用としてのみ提供されるものであり、以下に記載されている各用語の完全あるいは正式な定義、また各技術から派生するプライバシーやセキュリティ関連の事項についての完全あるいは正式な定義を提供するものではありません。

用語集

CA  認証局（CA）の項を参照。

Cookie  一部の Web サイトによってユーザのコンピュータに保存される小さな情報。Cookie を設定するサイトにアクセスすると、そのサイトはユーザのブラウザに、1 つまたは複数の Cookie をローカル ハード ディスク上に保存するよう要求します。ユーザが後でそのサイトに戻ったときに、ブラウザはそのサイトに属する Cookie を返信します。 Web サイトは、ショッピング カートの内容など、ユーザに関する情報を Cookie によって取得します。 ユーザは、Cookie に関する設定を行うことによって、Web サイトによる Cookie の使用方法や、Web サイトに対して保存を許可するユーザ個人情報の程度などを制御できます。外部 Cookie の項も参照。

Cookie マネージャ  Cookie の制御に使用できるブラウザの一部。詳しくは、「Cookie マネージャを使用する」を参照。

FIPS PUBS 140-1  「Federal Information Processing Standards Publications 140-1」の略で、暗号化モジュール、つまりデータの暗号化と復号化、あるいはその他の暗号処理（デジタル署名の作成や確認など）を実行するハードウェアまたはソフトウェアを実装するためのアメリカ合衆国政府の規格。アメリカ合衆国政府に対して販売される製品の多くは、FIPS 標準のいずれかの仕様に適合することが求められます。

FTP（File Transfer Protocol）  ネットワークを通じて、あるコンピュータから別のコンピュータにファイルを転送するための規格。 ブラウザでは、FTP を使用してファイルを取り込むことができます。

HTML（Hypertext Markup Language）  Web ページに使用されるドキュメント形式。 HTML 規格は、文字のレイアウト、フォント、スタイル、画像など、Web ページの構成要素を定義するためのタグ（コード）を規定するものです。

IMAP（Internet Message Access Protocol）  ユーザのすべてのメッセージおよび変更を、ローカル コンピュータのハード ディスクではなくサーバに保存し、任意のコンピュータからアクセスできるようにするプロトコル。 POP ではなく IMAP を使用することで、ディスク スペースを節約でき、送信済みトレイ、下書き、ユーザ設定フォルダなどを含むメール ボックス全体に任意の場所からアクセスすることができます。 IMAP サーバの場合、最初にダウンロードされるのはメッセージ ヘッダーだけなので、通常はモデム経由で IMAP サーバに接続する方が POP メール サーバに接続するよりも高速です。ISP の中には、IMAP をサポートしていないものもあります。

Java  Sun Microsystems によって開発されたプログラミング言語。 1 つの Java プログラムはさまざまな種類のコンピュータ上で使用できるため、各種コンピュータ用にそれぞれ異なるバージョンのプログラムを作成する必要がありません。 ブラウザは Java プログラムを自動的にダウンロードして実行できます。Java プログラムは、アプレットととも呼ばれます。

JavaScript  Web ページの構築に広く利用されているスクリプト言語。 フォームやボタンを表示するなど、Web ページをよりインタラクティブにするために使用されます。 JavaScript は Java と併用できますが、厳密には異なる言語であり、Java がなくても適切に動作します。

LDAP（Lightweight Directory Access Protocol）  複数のプラットフォームが企業アドレス帳などのディレクトリ サービスにアクセスするための標準的なプロトコル。 アドレス帳から LDAP ディレクトリにアクセスするようにブラウザを設定できます。 また、電子メール アドレスの自動補助機能に使用する目的で、Mail & Newsgroups が LDAP ディレクトリにアクセスできるように設定することも可能です。

Netscape 品質フィードバック エージェント  プログラムが突然終了してしまった場合などに技術的な情報を収集する、Netscape 7.0 に埋め込まれている小さなソフトウェア。 ユーザの了解を得た上で Netscape にその情報を送信します。 ユーザは、クラッシュ発生時の事情を説明するコメントを追加することもできます。 このようなフィードバックは、Netscape によってバグの修正やパフォーマンスの向上のために使用されます。 エージェントおよびその関連サービスは、トークバックとも呼ばれます。 詳しくは、オンライン ドキュメント「Netscape Quality Feedback System for Netscape（Netscape 品質フィードバック システム）」を参照。

OCSP（Online Certificate Status Protocol）  証明書を使用するたびに、証明書マネージャが電子メール証明書の有効性をオンラインでチェックするために使用する規則セット。このプロセスでは、特定の Web サイトで管理されている有効な証明書リストに対して、証明書のチェックが行われます。OCSP が動作するためには、コンピュータがオンライン状態でなければなりません。

P3P（Platform for Privacy Preferences）  アクセス先の Web サイトによるユーザ個人情報の用途をユーザがより詳細に制御できるようにすることを目的とした、W3C（World Wide Web Consortium）によって公開された規格。 この規格をサポートする Web サイトと Cookie の使用について詳しくは、「プライバシー レベルを設定する」を参照。規格そのものに関する情報については、オンライン ドキュメント「P3P Public Overview（P3P Public の概要）」を参照。

PKCS #11  スマート カードなどのセキュリティ デバイスを制御する公開キー暗号化規格。セキュリティ デバイス、スマート カードの項も参照。

PKCS #11 モジュール  PKCS #11 規格を使った暗号化と復号化などの暗号化サービスを管理する、ユーザのコンピュータ内にあるプログラム。PKCS #11 モジュール（暗号モジュール、暗号サービス プロバイダ、またはセキュリティ モジュールとも呼ばれる）は、ハードウェアまたはソフトウェアのいずれかに実装できる暗号デバイスのドライバと考えることができます。PKCS #11 モジュールは、常に 1 つまたは複数のスロットを管理します。スロットは、たとえば、スマート カードの読み込みなどの物理的なリーダーの 1 種として、またはソフトウェア内に実装できます。PKCS #11 モジュール用の各スロットには、同様にセキュリティ デバイス（トークンとも呼ばれる）を格納できます。セキュリティ デバイスは、実際に暗号化サービスを提供し、証明書やキーを保存できるハードウェアまたはソフトウェア デバイスです。証明書マネージャは、2 つの内蔵 PKCS#11 モジュールを提供しています。スマート カード リーダーやその他のハードウェア デバイスを制御するために、コンピュータも追加モジュールをインストールできます。

POP（Post Office Protocol）  ユーザのローカル コンピュータに新しいメッセージをダウンロードすることを要求する、標準的なメール サーバ プロトコル。ただし、サーバにコピーを残しておくことも可能です。 POP の場合、送信済みトレイ、下書き、ユーザ設定フォルダなどを含め、すべてのメッセージを 1 台のコンピュータ内に保存しなければなりません。 一方、IMAP はすべてのメッセージおよび変更を恒久的にサーバに保存し、任意のコンピュータからアクセスすることを可能にするプロトコルです。現在、ほとんどの ISP が POP をサポートしています。

SSL  SSL （Secure Sockets Layer）の項を参照。

SSL（Secure Sockets Layer）  認証と暗号化接続を確立するため、クライアントとサーバ間の相互認証を可能にするプロトコル。下位層は TCP/IP 上に構築され、HTTP、LDAP、IMAP、NNTP、およびその他のハイレベルのネットワーク プロトコルより下位で実行されます。TLS（Transport Layer Security）と呼ばれる新しい ITEF（Internet Engineering Task Force）規格は、SSL を基盤に作られています。認証、暗号化の項も参照。

TCP  TCP/IP（Transmission Control Protocol/Internet Protocol）の項を参照。

TCP/IP（Transmission Control Protocol/Internet Protocol   さまざまなオペレーティング システムを実行しているコンピュータ間を接続するための Unix プロトコル。 インターネットには不可欠なプロトコルで、国際基準となっています。

TLS  SSL（Secure Sockets Layer）の項を参照。

URL（Uniform Resource Locator）  Web 上にあるファイルまたはリソースの場所をブラウザに示すための規格化されたアドレス。例：http://www.netscape.com。ブラウザの場所ツールバーに URL を入力すると、対応する Web ページにアクセスできます。 また、URL は Web ページ内のリンクにも使用されており、そのリンクをクリックすることによって他の Web ページにアクセスできます。 URL は、インターネット アドレスまたは Web アドレスとも呼ばれます。

Web サイト  単一の企業、組織、個人などによって管理され、ハイパーリンクによってリンクされている、互いに関連する Web ページの集まり。 テキスト、グラフィック、オーディオ、ビデオなどのファイル、および他の Web サイトへのリンクなどで構成されます。

Web ページ  World Wide Web 上で固有のアドレス（URL）を持ち、テキスト、ハイパーリンク、グラフィックなどで構成される単一ドキュメント。

World Wide Web  Web とも呼ばれます。 Web サーバによって保存され、Web ブラウザと呼ばれるクライアント（Netscape Navigator など）によって表示される Web ページで構成されるインターネットの一部。

XML（Extensible Markup Language）  データ記述のためのオープン規格。 HTML とは異なり、Web ページ開発者が特別なタグを定義することが可能です。 詳しくは、W3C のオンライン ドキュメント「XML（Extensible Markup Language）」を参照。

XSLT（Extensible Stylesheet Language Transformation）  XML ドキュメントを他の XML ドキュメントまたは他の形式に変換するために用いられる言語。Netscape による XSLT のサポートについて詳しくは、「詳細設定 - 詳細」を参照。

暗号  暗号化のアルゴリズムの項を参照。

暗号化  情報の意味を第三者に理解できないようにスクランブルする処理。たとえば、コンピュータ間で接続を暗号化すると、第三者はそこでやり取りされている情報をアンスクランブル、すなわち復号化することが困難になります。暗号化された情報は、適切なキーを所有するユーザだけが解読できます。公開キー暗号化の項も参照。

暗号化アルゴリズム  暗号化や復号化などの暗号化処理を実行する際に使用される一連の規則や命令。暗号法とも言います。

暗号化キー  暗号化のためだけに使用される秘密キー。デュアル キー ペアは、暗号化キーとそれに対応する公開キー、および署名キーとそれに対応する公開キーから構成されています。

暗号化証明書  暗号化のためだけに使用される秘密キーに対応する公開キーを含む証明書。暗号化証明書は、署名処理には使用されません。デュアル キー ペア、署名証明書の項も参照。

暗号法  情報のスクランブル（暗号化）およびアンスクランブル（復号化）を行う技術とその実施のこと。たとえば、暗号化技術は、商業ベースの Web サイトとブラウザとの間で通信されるスクランブルされていない情報をスクランブルするのに使用します。公開キー暗号化の項も参照。

暗黙上の同意  黙示の同意または「オプトアウト」同意ともいいます。 Cookie やオンライン フォームなどの手段を通じてユーザに関する情報を収集することを Web サイトに対して許可するプライバシー設定を指します。 ユーザの同意を求めることなく情報収集が行われる場合もあります。

インターネット  TCP/IP などの標準的なプロトコルを使用して互いに通信する無数のコンピュータから成る、世界規模のネットワーク。 最初は 1969 年に米軍のために開発されたもので、次第に教育機関および研究機関が加わり、1990 年後半には莫大な数の企業、組織、および個人が利用するようになりました。 今日では、電子メール、World Wide Web のブラウズ、インスタント メッセージ、ユーザグループなど、多種多様な目的で使用されています。

インターネット プロトコル アドレス（IP アドレス）  TCP/IP ネットワークにおけるコンピュータのアドレス。 インターネット上のコンピュータは、それぞれ IP アドレスを持っています。 クライアントは、固定 IP アドレスまたはネットワークに接続するたびに動的に割り当てられる IP アドレスを持っています。 IP アドレスは、「204.171.64.2」のように 4 つの番号の組み合わせで表されます。

オブジェクト署名  ソフトウェア開発者が Java コード、JavaScript スクリプト、またはその他の各種ファイルに署名したり、ユーザが署名者を認識し、署名されたコードを使ったローカル システム リソースへのアクセスを制御したりすることを可能にする技術。

オブジェクト署名証明書  コード ファイルなどのオブジェクトへの署名に使用される対応秘密キーを持つ証明書。オブジェクト署名の項も参照。

外部 Cookie  あるサイトにアクセスしたときにユーザのコンピュータに保存される、別のサイトからの Cookie。 Web サイトによっては、他の Web サイトがホストするコンテンツを表示するものがあります。 このようなコンテンツには、画像、テキスト、宣伝などさまざまなものがあり、ユーザが直接アクセスしていないにもかかわらず、この外部 Web サイトもユーザのコンピュータに Cookie を保存することが可能です。 外部 Cookie は、第三者 Cookie ともいいます。

カレンダー  スケジュールや関連タスクを把握するためのアプリケーション（Netscape Calendar）。 業務用に Netscape を使用している場合は、Netscape の［ウィンドウ］メニューから使用できる可能性があります。 家庭で Netscape を使用している場合、または勤務先で Netscape Calendar が提供されていない場合は、www.netscape.com にアクセスして Web ベースのカレンダーを使用できます。

キー  データを暗号化または復号化するために暗号化アルゴリズムで使用される大きな数値。たとえば、あるユーザの公開キーを使って、そのユーザに送信するメッセージを他のユーザが暗号化することができます。暗号化したメッセージを解読するには、対応する秘密キーを使用しなければなりません。公開キー暗号化の項も参照。

キャッシュ  コンピュータのハード ディスクまたはランダム アクセス メモリ（RAM）に保存された、いくつかの Web ページのコピー。 Web をブラウズすると、ブラウザによってこれらのコピーが収集されます。 キャッシュにすでにコピーが保存されている Web ページへのリンクをクリックしたり URL を入力したりすると、ブラウザは元のページとキャッシュ内のコピーとを比較し、変更がなければ元のページの代わりにコピーを表示することによって、処理およびダウンロードに必要な時間を節約します。

クライアント  サーバにリクエストを送信したり、サーバからの情報を受信したりする、ブラウザなどのソフトウェア。通常、サーバは別のコンピュータ上で実行されています。クライアント ソフトウェアを実行しているコンピュータもクライアントと言います。

クライアント SSL 証明書  SSL（Secure Sockets Layer ） プロトコルを使って、ブラウザ ソフトウェアなどのクライアントが、クライアントの ID 情報（または、クライアントを使用するユーザの個人情報）を認証するために、サーバに提供する証明書。クライアント認証の項も参照。

クライアント認証  サーバに対してクライアントを識別する処理。識別には、名前とパスワード、またはクライアント SSL 証明書とデジタル署名されたデータなどが使用されます。SSL（Secure Sockets Layer）、サーバ認証の項も参照。

検索エンジン  ユーザが World Wide Web で特定の情報を検索し、取得することを可能にする Web ベースのプログラム。 Web ドキュメントに含まれるテキストや一連のキーワードを検索したり、Web ドキュメントを閲覧してそれを手作業で索引化するライブラリアンを利用したりします。 通常、ユーザが検索ボックスに単語や語句を入力すると、検索エンジンは該当する Web ページへのリンクを表示します。この単語や語句の入力は、クエリーとも呼ばれます。

公開キー  公開キー暗号化で使用されるキー ペアの片方。公開キーは、証明書の一部として自由に配布および公開されます。一般に、公開キーは公開キーの所有者に送られるデータを暗号化するために使用されます。後で暗号化されたデータを受け取った公開キーの所有者は、対応する秘密キーを使ってそのデータを解読します。

公開キー インフラストラクチャ（PKI）  ネットワーク環境における公開キー暗号化と証明書の使用をサポートする規格とサービス。

公開キー暗号化  エンティティ（個人、組織、またはルータのようなハードウェアなど）がその身元を電子的に証明したり、電子データに署名して暗号化したりするために広く用いられている技術とその規格。公開キー暗号化には、公開キーと秘密キーという 2 つのキーがあります。公開キーは、証明書の一部として公開されます。証明書は、公開キーを特定の個人情報に関連付けます。対応する秘密キーは他人に公開されずに維持されます。公開キーで暗号化されたデータは、対応する秘密キーを使わないと解読できません。

コマンド ツールバー  ブラウザ ウィンドウの上部に位置する、［戻る］ボタンや［次へ］ボタンを含むツールバー。

コンポーネント バー  Netscape ウィンドウの左下部分に位置するツールバー。 Navigator、Mail & Newsgroups、Instant Messenger などの Netscape コンポーネントのアイコンがあり、クリックすることによってコンポーネントを切り替えることができます。

サーバ  クライアントからの要求を受け取ったり、クライアントに情報を送ったりするソフトウェア（Web ページを提供するソフトウェアなど）。通常、クライアントは別のコンピュータ上で実行されています。サーバ ソフトウェアが実行されているコンピュータもサーバと呼びます。

サーバ SSL 証明書  SSL （Secure Sockets Layer）プロトコルを使用してサーバの識別情報を認証するために、サーバがクライアントに提供する証明書。

サーバ認証  サーバ SSL 証明書を使用し、クライアントに対してサーバを識別する処理。クライアント認証、SSL（Secure Sockets Layer）の項も参照。

サブジェクト  証明書が識別する個人、組織、またはルータなどのエンティティ。特に、証明書のサブジェクト フィールドには、証明対象のサブジェクト名やその他の特性が保存されます。

サブジェクト名  証明書のサブジェクトを記述する識別名（DN）。サブジェクトごとに固有の識別名が使用されます。

識別名（DN）  証明書のサブジェクトを一意に識別する、特殊な形式の名前。

従属認証局  他の従属認証局またはルート認証局によって署名される証明書を持つ認証局。証明書チェーン、ルート認証局の項も参照。

証明書  デジタル版の身分証明書に相当するもの。証明書には、個人、会社、またはその他の事業団体の名前が記されており、証明書に含まれている公開キーがそのエンティティに属していることを証明します。メッセージやその他のデータにデジタル署名機能を使用すると、証明書に含まれている公開キーに対応する秘密キーを使って、そのメッセージ用のデジタル署名が作成されます。証明書は、認証局が発行し、デジタル署名されます。証明書の有効性は、認証局のデジタル署名をチェックして確認できます。デジタル ID、デジタル パスポート、公開キー証明書、X.509 証明書、またはセキュリティ証明書ともいいます。公開キー暗号化の項も参照。

証明書チェーン  連続する認証局によって署名された一連の証明書の階層構造。認証局証明書は、認証局を識別するためのもので、認証局によって発行された証明書に署名するために使用されます。認証局証明書は、順次、その親 CA の認証局証明書によって署名されます。このプロセスは、ルート認証局までさかのぼって行われます。

証明書の確認  証明書マネージャは、証明書を確認する際、その認証局独自の認証局証明書があること、およびその種の証明書の発行に関して「信頼できる」認証局によってデジタル署名が作成されていることを確認します。また、確認しようとしている証明書が「信頼できない」とマークされていないかどうかも確認します。その後、OCSP（Online Certificate Status Protocol）がオンになっている場合は、オンライン チェックも行われます。このオンライン チェックとは、証明書自体またはブラウザの確認設定のいずれかで指定された URL に保存されている有効な証明書のリストで証明書を検索することです。これらのいずれかのチェックに失敗すると、証明書マネージャは証明書を確認不可と判断し、個人情報を認識しません。

証明書の更新  間もなく有効期限が切れる証明書を更新する処理。

証明書破棄リスト（CRL）  認証局（CA）が作成および署名した証明書のうち、すでに破棄された証明書のリスト。証明書を認証用に使用する前に、最新の CRL をブラウザまたはサーバにダウンロードし、証明書が有効かどうか確認します。

証明書バックアップ パスワード  バックアップ中またはバックアップ済みの証明書を保護するパスワード。証明書をバックアップしようとすると、パスワードを設定するよう指示されます。バックアップ済みの証明書を復元する際には、そのパスワードの入力が求められます。

証明書フィンガープリント  証明書に関連付けられた固有の番号。この番号は証明書自体には含まれず、証明書の内容に関数を適用して生成されます。証明書の内容を 1 文字でも変更すると、関数によって別の番号が生成されます。このため、証明書フィンガープリントを使って、証明書が変更されていないかどうかを確認できます。

証明書ベースの認証  証明書と公開キー暗号化を基にして個人情報を確認すること。パスワードベースの認証の項も参照。

証明書マネージャ  証明書の表示および管理をするためのブラウザの一部。証明書マネージャのメイン ウィンドウを表示するには、［編集］メニューから［設定］を選択し、［プライバシーとセキュリティ］設定の［証明書］パネルで［証明書の管理］をクリックします。

署名キー  署名のためだけに使用される秘密キー。デュアル キー ペアは、署名キーに対応する公開キーと、暗号化キーに対応する公開キーから構成されています。

署名証明書  受取人が差出人の個人情報を確認できるように、転送データの署名に使用される対応秘密キーの証明書。認証局は、多くの場合、電子メール メッセージの暗号化に使用する暗号化証明書の発行と同時に、電子メール メッセージの署名に使用する署名証明書を発行します。デュアル キー ペア、デジタル署名の項も参照。

信頼  個人ユーザやその他のエンティティを信用すること。通常、公開キー インフラストラクチャ（PKI）において信頼とは、証明書のユーザと証明書を発行した認証局との関係を表します。証明書マネージャを使って、ある認証局を信頼するように設定すると、証明書マネージャは、証明書ごとに設定を変更しない限り、その認証局が発行した有効な証明書をすべて信頼します。証明書マネージャの［認証局］タブを使って、特定の認証局が発行した証明書のうち、信頼できる証明書と信頼できない証明書の条件を指定できます。

ステータス バー  各 Netscape ウィンドウの最下部に位置するツールバー。 左側にはコンポーネント バーがあり、右側には各種ステータス アイコンがあります。 たとえば、Navigator にはロック アイコンなどのステータス アイコンがあります。

スマート カード  通常、マイクロプロセッサを搭載したクレジット カード サイズの小型デバイス。日本では IC カードと呼ばれます。スマート カードには、キーや証明書などの暗号情報が保存され、暗号処理を実行する機能が備わっています。スマート カードは PKCS #11 規格を使用しています。これは、セキュリティ デバイスの一種です。

スロット  PKCS #11 モジュールによって制御され、セキュリティ デバイスを格納できるように設計されているハードウェアの一部、またはソフトウェア内のそれに相当する部分。

セキュリティ デバイス  暗号化と復号化などの暗号サービスを提供し、証明書やキーを保存するハードウェアまたはソフトウェア。ハードウェアに実装されるセキュリティ デバイスの 1 つにスマート カードがあります。証明書マネージャには、ブラウザを実行するときに常に有効になる ソフトウェア セキュリティ デバイス というセキュリティ デバイスが組み込まれています。各セキュリティ デバイスは、それぞれ専用のマスター パスワードで保護されています。

セキュリティ トークン  セキュリティ デバイスの項を参照。

セキュリティ モジュール  PKCS #11 モジュールの項を参照。

セキュリティ証明書  証明書の項を参照。

ソフトウェア セキュリティ デバイス  ユーザの証明書に関連付けられた秘密キーを保存するために証明書マネージャが使用する、標準設定のセキュリティ デバイス。ソフトウェア セキュリティ デバイス には、秘密キーの他に電子メール パスワード、Web サイト パスワード、その他の機密情報を暗号化するためにパスワード マネージャが使用するマスター キーも保存されます。秘密キー、マスター キーの項も参照。

第三者 Cookie  外部 Cookie の項を参照。

対称暗号化  メッセージの暗号化と復号化に 1 つの暗号キーを使用する暗号化方式。

タンパ検出  電子形式で受信したデータが変更されていないこと、つまり、受信したデータが元のデータに完全に一致していることを確認するしくみ。

デジタル ID  証明書の項を参照。

デジタル署名  署名されるデータと署名者の秘密キーの両方から作成されるコード。このコードはデータごとに新しいものが作成され、同じコードが作成されることはありません。メッセージにカンマが 1 つ追加されても、そのメッセージのデジタル署名は変わります。適切なソフトウェアによってデジタル署名が問題なく確認されると、ユーザがトランザクションまたはメッセージを承認したことが証明されるだけでなく、デジタル署名された後にデータが変更されていないことも証明されます。デジタル署名は手書きの署名とは何の関係もありませんが、法律上、同様の目的で使用されることもあります。否認防止、タンパ検出の項も参照。

デュアル キー ペア  2 つの異なる証明書に対応する公開キーと秘密キーのペア 2 つ、合計 4 つのキー。一方のペアの秘密キーは署名処理に、もう一方のペアの公開キーと秘密キーは暗号化と復号化処理に使用されます。各ペアは、それぞれ個別の証明書に対応しています。公開キー暗号化の項も参照。

盗聴  ネットワーク上で送信された情報が、対象以外の企業や個人によって不正に傍受されること。

トークバック  Netscape 品質フィードバック エージェント の項を参照。

トークン  セキュリティ デバイスの項を参照。

なりすまし  他者のふりをすること。たとえば、ユーザが実際には存在しない jdoe@mozilla.com という電子メール アドレスを持つユーザのふりをしたり、コンピュータが実際には存在しない www.mozilla.com というサイトのふりをすることです。なりすましは、不当表示の一種です。

認証  パスワード、証明書、個人識別番号（PIN）、またはその他の情報を使って、コンピュータ ネットワーク上で個人情報を確認すること。パスワードベースの認証、証明書ベースの認証、クライアント認証、サーバ認証の項も参照。

認証局（CA）  個人や企業の ID 情報を確認し、証明書を発行する機関。認証局は、証明書の更新や破棄、破棄された証明書リストの定期的な作成も行います。認証局は、独立したベンダー（クライアント証明書に記載された CA など）の場合もあれば、証明書を発行するサーバ ソフトウェア（Mozilla 証明書管理システムなど）を使用している個人や団体の場合もあります。証明書、証明書破棄リスト（CRL）の項も参照。

認証局証明書  認証局を識別する証明書。認証局(CA)、従属認証局、ルート認証局の項も参照。

場所ツールバー  Navigator ウィンドウの上部に位置する、URL や検索用語を入力できるフィールド（および関連ボタン）。詳しくは、「別のページに移動する」を参照。

パスワード マネージャ  ユーザ名とパスワードをコンピュータのハード ディスク上に保存し、必要に応じてそれを自動入力するブラウザ機能の一部。詳しくは、「パスワード マネージャを使用する」を参照。

パスワードベースの認証  名前とパスワードを使用した、信頼性の高い識別方法。認証の項も参照。

否認防止  メッセージ送信者がメッセージを送信したことを否定できないようにすること。通常の手書きによる署名は否認防止形式の 1 つです。デジタル署名もその 1 つです。

秘密キー  公開キー暗号化で使用されるキー ペアの片方。秘密キーは他人に公開されずに維持され、対応する公開キーで暗号化されたデータを解読するために使用されます。

フィンガープリント  証明書フィンガープリントの項を参照。

フォーム マネージャ  オンライン フォームに入力した、名前、アドレス、電話番号などの個人データの保存を可能にするブラウザの一部。いったんデータを保存した後は、Web サイトにフォームが表示されるたびに、フォーム マネージャによって自動的に記入が行われます。詳しくは、「フォーム マネージャを使用する」を参照。

復号化  暗号化されたデータをアンスクランブルする処理。暗号化の項も参照。

ブックマーク  ユーザ設定ツールバー内のブックマーク アイコンをクリックするか、［ブックマーク］メニューからブックマーク名を選択することによって簡単にアクセスできる、保存された Web ページ アドレス（URL）。

不当表示  個人または組織を実際とは異なるものであると称すること。たとえば、ある Web サイトを家具店の HP のように見せて、実際にはクレジット カードによる支払いを行わせるだけで、商品を配送しない場合などがあります。なりすましの項も参照。

プラグイン  オーディオ クリップやビデオ クリップを再生する機能など、新しい機能をブラウザに提供するヘルパー アプリケーションの一種。 他のヘルパー アプリケーションとは異なり、プラグインは、メイン ブラウザのインストール ディレクトリ内にあるプラグイン用ディレクトリにインストールされ、通常はブラウザ内で開くことができます。 たとえば、オーディオ プラグインは、Web ページや電子メール メッセージに含まれるオーディオ ファイルの再生を可能にします。 プラグイン アプリケーションの例として、Macromedia Flash Player や Java などがあります。

プロキシ  他のクライアントの代理として要求を出すために、サーバとクライアントの両方の役割を担う中間プログラムまたは仲介プログラム。

ヘルパー アプリケーション  ブラウザによってダウンロードされたファイルを開くまたは表示するために使用されるアプリケーション。 特殊な種類のヘルパー アプリケーションとして、プラグインがあります。プラグインは、メイン ブラウザのインストール ディレクトリ内にあるプラグイン用ディレクトリにインストールされるヘルパーアプリケーションで、通常はブラウザ内部で開くことができます。 Microsoft Word や Adobe Photoshop などの外部アプリケーションもヘルパー アプリケーションですが、これらのアプリケーションは、ブラウザのインストール ディレクトリにインストールされないためプラグインではありませんが、ダウンロード ダイアログ ボックスから開くことができます。

ホーム ページ  ブラウザを起動したり、［ホーム］ボタンをクリックするたびに表示されるよう設定されているページ。 また、Web サイトのメインのページを指すこともあります。Web サイトのホーム ページからは、サイトの他の部分にアクセスできます。

マスター キー  証明書マネージャが情報の暗号化に使用する対称キー。たとえば、パスワード マネージャは証明書マネージャとマスター キーを使用して、電子メールのパスワード、Web サイトのパスワードなど、保存されている機密情報を暗号化します。対称暗号化の項も参照。

マスター パスワード  セキュリティ デバイスに保存されているマスター キーまたは秘密キーを保護するために、証明書マネージャが使用するパスワード。証明書マネージャは、たとえば、ユーザが電子メール メッセージに署名したり、Web サイトに対して本人であることを証明したりするためにユーザ独自の証明書を使用する場合、ユーザの秘密キーにアクセスする必要があります。パスワード マネージャまたはフォーム マネージャが個人情報を読み取ったり追加したりする場合、証明書マネージャはユーザのマスター キーにアクセスする必要があります。マスター パスワードの設定や変更は、［マスター パスワード］設定パネルで行います。セキュリティ デバイスごとに異なるマスター パスワードを設定してください。秘密キー、マスター キーの項も参照。

ユーザ設定ツールバー  標準設定で Navigator の場所ツールバーのすぐ下に表示されるカスタマイズ可能なツールバー。 追加および削除が可能な［ホーム］、［検索］、［ブックマーク］などの標準ボタンがあります。 また、お気に入りのブックマーク用のボタンや、ブックマークのグループを含むフォルダ用のボタンを追加することもできます。詳しくは、「ユーザ設定ツールバー」を参照。

ルート認証局  証明書チェーンの最上部に位置する、自己署名付き証明書を持つ認証局。従属認証局の項も参照。
