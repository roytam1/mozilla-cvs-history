ブラウザから Web サイトに送られる情報

（たとえばリンクをクリックしたり、URL を入力して）ブラウザに Web ページが表示されたときや、電子メール メッセージに Web ページが表示されたときに、ブラウザからそのサイトに特定の情報が送られます。 この情報には、ユーザの操作環境やインターネット アドレス、またどのページからアクセスしてきたかなどの情報が含まれることがあります。

 

操作環境

サイトには、ブラウザの種類やオペレーティング システムなどユーザの操作環境についての情報が送信されます。 これらの情報は、ユーザの画面上に最適なページを表示するのに役立ちます。 たとえば、オペレーティング システムが Windows 2000 で、Netscape 6.2.3 のフランス語バージョンを使用しているなどの情報がサイトに送信されます。

 

インターネット アドレス

ユーザが要求しているページを正しい場所に送信するために、サイトはブラウザからインターネット アドレス（インターネット プロトコルまたは IP アドレス）を取得する必要があります。 IP アドレスがなければ、サイトはユーザから要求されたページを表示できません。

IP アドレスは、仮 IP アドレスでも固定（静的）IP アドレスでも構いません。

電話線に接続されている標準モデムを使用してインターネットに接続している場合は、ログオン時にインターネット サービス プロバイダ（ISP）から仮 IP アドレスが割り当てられている可能性があります。 インターネット セッション中（たとえば、サインオフまたダイヤルアップ接続を切断するまで、またはインターネットへの接続を終了するまで）、ユーザはその割り当てられた仮 IP アドレスを使用します。 各 ISP は多くの IP アドレスを所有しており、それらの IP アドレスを任意にユーザに割り当てます。

DSL、ケーブル モデム、または光ファイバ接続を利用している場合は、毎回の接続時に使用する固定 IP アドレスが割り当てられている可能性があります。

IP アドレスは電子メール アドレスと同じものではありません。

 

参照元のページ

サイトには、リンクをクリックしたときにユーザがどのページを表示していたかについての情報も送信されます。 これにより、ユーザがどのサイトからジャンプしてきたかが明らかになります。 また、ユーザがサイト内でジャンプした場合には、どのページからジャンプしてきたかが分ります。

 

Cookie の概要と機能

Cookie は一部のサイトで使用される小さな情報です。 Cookie を使用するサイトにアクセスすると、そのサイトはユーザのブラウザに対し、1 つまたは複数の Cookie をハード ディスク上に配置してもよいかどうかを尋ねます。

後で、ユーザがそのサイトに再びアクセスすると、ブラウザはそのサイトに関連する Cookie を返信します。

Cookie の標準設定を使用している場合は、このアクティビティがユーザの目に触れることはありません。サイトが Cookie を設置したり、ブラウザがサイトに Cookie が送信したりする操作は、ユーザの知らないうちに行われます。 ただし、Cookie が設定される前に確認のメッセージを表示するように設定することができます。 設定の方法については、「プライバシー レベルを設定する」を参照してください。

 

Cookie 情報の使用目的

サイトは Cookie を使用してユーザの過去のアクセスに関する情報を得ることができます。 たとえば、あるユーザが現地の天気予報を検索し、オンラインで書籍を購入したとします。すると、そのサイトは Cookie を使ってユーザの住んでいる都市や好きな作家を記憶し、次回ユーザがサイトにアクセスしたときにその情報を活用します。

多くの Web サイトには、収集した情報の使用目的を説明したプライバシー証書があります。

Web サイトのプライバシー証書を読むと、収集する情報やその使用目的、その情報の使用に関してユーザに与えられた選択などを確認できます。 詳しくは、「プライバシー証書を表示する」を参照してください。

公開されているプライバシー証書を使って Cookie を管理する方法については、「プライバシー レベルを設定する」を参照してください。

 

第三者の Cookie について

サイトの Cookie が設置された場合、その Cookie はそのサイトに対してのみ返信されます。 あるサイトによって設定された Cookie が別のサイトに送られることはありません。 Web サイトは、独自の Cookie しか受け取ることができないため、ユーザがそのサイトにアクセスしたときの行動を学習することはできますが、そのユーザの一般的なインターネット アクティビティについての情報を得ることはできません。

しかし、あるWeb サイトに、他のサイトでホストされているコンテンツが表示されることがあります。 そのコンテンツとは、画像やテキスト、あるいは広告などです。 そのようなエレメントをホストしている別の Web サイトは、ユーザが直接そのサイトにアクセスしなくても、ユーザのブラウザに Cookie を設置することができます。

このようにユーザがアクセスしているサイト以外のサイトによって設置される Cookie のことを、第三者 Cookie または外部 Cookie と言います。 Web サイトでは、透過 GIFと呼ばれる特殊画像を伴なう第三者 Cookie を使用することがあります。この透過 GIF は、ユーザ数のカウントや電子メール応答の記録に利用されるだけでなく、サイトの利用状況を分析したり、Web サイトの機能をカスタマイズするのに役立ちます （透過 GIFは Web ビーコンまたは Web バグとも呼ばれます）。

通常の Cookie だけを受け入れて第三者 Cookie を拒否するように、Cookie の設定を調整することができます。

 

電子メール メッセージにおける Web ページのコントロール方法

電子メール メッセージの一部として受信する Web ページの Cookie、画像、JavaScript、プラグインは無効にすることができます。

これらの機能の一部またはすべてを有効にしておくと、Web をブラウズするときに便利なこともありますが、メッセージに添付された Web ページには不要な場合もあります。

権限のない第三者から個人情報を保護する方法

個人情報のプライバシーを守るには、アクセスするサイトや使用するインターネット サービスのプライバシー証書を確認し、オンラインで個人情報を提供する場合には十分な注意を払う必要があります。

インターネットは公共のネットワークです。 名前や電話番号、住所などの個人情報を（Web ページ、電子メールなどで）ネットワークを介して送信すると、他人に傍受される可能性があります。

以下に、Web サイトのプライバシー証書で確認するとよい事項をいくつか紹介します。

  • どのような個人情報が収集されるか
  • 収集された個人情報がどのように使用されるか
  • 収集された情報が別のサイトと共有されるか、また共有される情報の用途に関する選択肢がユーザに与えられるか
  • ユーザが自分の個人情報の一部またはすべてにアクセスできるか（内容を確認または更新するため）
  • 収集された個人情報がどのように保護されるか
  • 質問や問題が発生した場合の Web サイトへの連絡方法が記載されているか

Web サイトのプライバシー証書を確認する方法については、「プライバシー証書を表示する」を参照してください。
