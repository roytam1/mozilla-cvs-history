Localization Notes for CONFIG_PREFS

These entries make up the configurable Guide Toolbar and Help menu items and their functions. As part of the Unlimited Localization Program, you may define your own set of Guide Toolbar and Help Menu items.  You MUST remove or replace with your own links the following Help Menu items:
  • Release Notes
  • Product Information and Support
  • Software Updates
  • Register Now
  • Member Services
CONFIG_PREF items are identified in the localization notes as "CONFIG_PREFSRCDATA" followed by the Javascript function name.  For example,
    CONFIG_PREFSRCDATAtoolbar.places.default_url
Most items in this list of preferences consist of two related components: a label and a URL.  To localize a label, only edit the portion of the preference that are within the second set of quotes.  For example,
  • config("toolbar.places.item_0.label","The Internet");
The words, "The Internet" are contained in the second set of quotes; that is the part you localize.  Because these are button labels, you should try to keep them as short as possible.  If you find that you must use a long word or phrase in your translation, you may discover that the buttons no longer display the entire text.  You can increase the maximum number of characters the Toolbar buttons will display by increasing the value of the browser.personal_toolbar_button.max_chars preference.

You can use the "-" character in a label to create a separator.

You can specify a sub-folder in the Guide Toolbar by specifying the URL to be the word "FOLDER".  For example:

  • config("personal_toolbar.item_1.url","FOLDER");
This lets the code that lays out the toolbar know that the current item is a folder with sub-folders.  DO NOT localize the word "FOLDER".

Forbidden CONFIG_PREFS Resources

There are some URLs in CONFIG_PREFS which must not be changed in any way.  The first kind is the about: page URLs:
  • config("menu.help.item_12.url","about:plugins");
  • config("menu.help.item_13.url","about:fonts");
You may translate the associated labels but not these URLs. (You can, of course, delete them entirely, but then you won't have access to these "about" pages via the menus.)  Refer to the Localizing URLs section for more information.
Finally, some items in CONFIG_PREFS are non-localizable Javascript preference values.  For example,
  • config("toolbar.logo.frames",-1);
  • config("personal_toolbar.CreateNewToolbar", false);
  • config("personal_toolbar.NewToolbarFolderName", "");
Don't alter these.
