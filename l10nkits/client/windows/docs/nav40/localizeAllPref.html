Localization Notes for ALL_PREFS

These preferences are identified in the localization notes as "ALL_PREFSRCDATA" followed by the Javascript function name for that preference.  For example:
    ALL_PREFSRCDATAnetwork.search.url
The localizable portion of this preference will appear in the XL_STR field.  If this value is a string, it will be surrounded with quotes.  If it is a number, it will not.  Do not delete quotes if they are present and do not add them if they are absent!  Many entries also begin with tab characters or initial spaces; it is best not to alter these, as well.

NOTE: the following section is for reference, showing the way the complete Javascript syntax looks.  The entries contained in the translation databases separate the Javascript function from the data that function returns. Comments are reproduced in the COMMENT field of the translation database where appropriate.

You only need to localize the following entries. The parts you need to change are highlighted in majenta. Comments are in green. Localize URLs according to the rules.

  • pref("network.search.url", "http://cgi.netscape.com/cgi-bin/url_search.cgi?search=");
      • This URL is connected to the Navigator's Search button.
       
  • pref("browser.startup.homepage", "http://home.netscape.com/");
      • This is the default home page value that is attached to the Home button and the Home menu item.
  • pref("browser.personal_toolbar_button.max_chars", 15);
      • This determines the maximum number of characters a personal toolbar button title can contain.  You may not need to change this, but it's there if you do.
       
  • pref("intl.accept_languages",    "en");
      • This determines the default value of the ACCEPT_LANG property of the META tag. You can see this value by choosing Edit | Preferences, select Navigator | Language; the default value will be the 2 letter ISO Language code value you supply here.
       
  • pref("intl.accept_charsets",  "iso-8859-1,*,utf-8");
      • Replace iso-8859-1 with sjis for Japanese, euc-kr for Korean, gb for Simplified Chinese, Big5 for Traditional Chinese, iso8859-2 for Latin 2 languages.
       
  • pref("intl.character_set",  2);
      • You only need to change this for non-LATIN1 languages.  Use the CSID Reference table to look up the appropriate value for other languages.
       
  • pref("intl.font_encoding", 6);
      • maybe not used, 6 = CS_MAC_ROMAN, 1292 = ko, 263 = trad. chinese, 264 = simp. chinese
     
  • pref("editor.template_location", http://home.netscape.com/home/gold3.0_templates.html");
  • pref("editor.default_template_location", "http://home.netscape.com/home/gold3.0_templates.html");
      • Localize these URL according to the rules.
  • pref("editor.html_directory",       "C:\\My Documents");
  • pref("editor.image_directory",       "C:\\My Documents");
      • If the My Documents directory is translated on the localized version of Windows you working on, then match this preference with the localized value.
       
  • pref("editor.color_scheme",     "Netscape Default Colors");
      • This string should match the value used to translate STRINGTABLE42169 (but it's not critical if it doesn't).
       
  • pref("ldap.directory1.description", "Personal Address Book");
  • pref("ldap_1.directory1.description", "Personal Address Book");
      • These entries appears as the description for searching the personal address book in a mail directory search.  Translate them the way you do all the other instances of this string.
       
  •  pref("netcaster.containers.container1", "-3;'Channel_Finder'; 'http://netcaster.netscape.com/finder/container/index.html'; 1440;0");
      • Localize this URL according to the rules.
